# G4 · 文件存储与导入流水线

**版本：** v1.0
**日期：** 2026‑04‑21
**适用对象：** 后端 / 运维 / 数据
**地位：** 支撑云霓采集的语音/图片、造浪池的文档"炸开"、冰山的导出文件三类场景。

---

## 一、技术选型

| 组件 | 选型 | 理由 |
| :--- | :--- | :--- |
| 对象存储 | **S3-兼容**（生产用 AWS S3 / 自建用 MinIO） | 行业标准、成本可控 |
| 异步任务 | **Celery + Redis** | Python 生态成熟 |
| 文档解析 | **Unstructured** + **PyMuPDF** + **python-docx** | 多格式覆盖 |
| ASR | **Whisper (OpenAI API 或本地 large-v3)** | 中文识别质量高 |
| OCR | **PaddleOCR** | 中文优势 |
| 元数据库 | **PostgreSQL `assets` 表**（见 G1） | 强一致 |

---

## 二、桶 (Bucket) 规划

| Bucket | 用途 | 生命周期 | 加密 |
| :--- | :--- | :--- | :--- |
| `qp-uploads-raw` | 原始上传（语音、图片、PDF） | 90 天后冷归档；180 天后删（除非引用） | SSE-S3 |
| `qp-processed` | 处理后产物（OCR 文本、ASR 转写、Chunk） | 与父 asset 同期 | SSE-S3 |
| `qp-exports` | 用户导出文件 | 30 天后删 | SSE-S3 + 客户端密钥（可选） |
| `qp-snapshots` | 大快照 JSON | 与冰山同期；冰山删则删 | SSE-S3 |
| `qp-public-shares` | 公开链接静态资源 | 链接过期后立即删 | 公共读 |

**对象命名约定：**
```
<bucket>/<user_id>/<yyyy>/<mm>/<asset_id>.<ext>
例: qp-uploads-raw/u-1234/2026/04/a-abcd.mp3
```

## 三、上传流程（直传，绕过 API 网关）

```
[Client]
  │ 1. POST /api/v1/assets/upload-token  (只声明 mime/size/sha256)
  ▼
[Backend] → 创建 asset(status=UPLOADING) → 签发预签名 PUT URL
  │
  ▼
[Client] → 直接 PUT 到 S3
  │
  ▼
[S3 Event] → SQS / Webhook → [Backend] confirm-upload
  │
  ▼
[Backend] → asset.status = READY → 触发 Celery 任务（如需处理）
```

### 3.1 上限控制

| 类型 | 单文件上限 | 用户日上限 |
| :--- | :--- | :--- |
| 图片 | 20 MB | 200 张 |
| 语音 | 50 MB / 30 分钟 | 100 段 |
| 文档（PDF/DOCX） | 100 MB | 50 份 |
| 视频（导出回写） | 500 MB | 10 段 |

超限返回：`429 Too Many Requests`，文案"**水位过高，请稍候。**"

### 3.2 病毒扫描

所有上传后异步过 ClamAV，命中则 `status = REJECTED`，资产标记隔离。

## 四、文档"炸开"流水线

### 4.1 流水线图

```
PDF/DOCX/MD upload
    │
    ▼
[ParseTask]  (Celery, queue="parse")
  - PDF: PyMuPDF 提文本 + 页码
  - DOCX: python-docx
  - MD: 直接读
    │
    ▼
[ChunkTask]  分块 (≤ 500 token, 50 token overlap)
    │
    ▼
[ConceptExtractTask]  调 LLM (G5 §炸开 prompt) 提概念
    │
    ▼
[NodeMaterializeTask]  写入 Neo4j as Node(type=DOC_CHUNK)
    │
    ▼
[EmbedTask]  生成 embedding，写入向量索引
    │
    ▼
[RelateTask]  AI 探测 RELATES_TO 关系
    │
    ▼
[NotifyTask]  WebSocket 推送 "炸开完成 N 节点"
```

### 4.2 错误重试

- 每个 Task 最多 3 次重试，指数退避 (1s/10s/60s)
- 第 4 次失败：写入 `failed_tasks` 表 + 告警
- 用户侧文案："**这片水域过深，AI 一时探不到底。**" + 重试按钮

### 4.3 进度反馈

WebSocket 进度事件：
```json
{
  "type": "DOC_EXPLODE_PROGRESS",
  "asset_id": "...",
  "phase": "PARSING | CHUNKING | EXTRACTING | MATERIALIZING | EMBEDDING | DONE",
  "progress": 0.62
}
```

## 五、ASR 与 OCR 流水线

### 5.1 语音转写
- 文件 ≤ 30s：同步走，结果直接返回
- 文件 > 30s：异步 Celery，完成后回调更新对应 Node `content`
- 调用 OpenAI Whisper API（成本敏感场景换本地）
- 转写文本作为 Node content；原音频留作 asset 关联

### 5.2 图像 OCR
- 端侧优先（移动端 ML Kit / Vision）
- 端侧失败 / 用户明确"高质量识别" → 上传至 PaddleOCR 服务
- 输出文本写入 Node content，原图作为 asset 关联

## 六、生命周期与垃圾回收

### 6.1 孤儿资产清理

每日 Job：
```sql
-- 找出 24 小时内无任何 Node 关联且状态仍为 UPLOADING 的资产
DELETE FROM assets
WHERE status = 'UPLOADING' AND created_at < now() - INTERVAL '24 hours';

-- 找出已被所有引用 Node 物理删除的 READY 资产
UPDATE assets SET status = 'DELETED', deleted_at = now()
WHERE id IN (
  SELECT a.id FROM assets a
  LEFT JOIN node_asset_refs r ON r.asset_id = a.id
  WHERE r.id IS NULL AND a.status = 'READY'
);
```

物理删除 S3 对象的二级 Job 在 7 天后执行（防误删）。

### 6.2 用户注销

GDPR 风格的"被遗忘权"：
1. 用户提交注销 → `users.deleted_at = now()`
2. 30 天宽限期可恢复
3. 满 30 天后：所有用户资产、节点、湖泊（仅 OWNER 时）打包导出邮寄一份给用户，然后物理删除

## 七、安全

1. 预签名 URL 有效期：上传 15 分钟，下载 5 分钟
2. 下载需鉴权：除 `qp-public-shares` 外，所有读取走签名 URL（不直接公开桶）
3. 跨用户访问：URL 中的 `user_id` 必须与 JWT `sub` 一致
4. EXIF 清洗：图片上传后剥离 GPS / 设备指纹
5. PDF 解析沙箱：用 `pdfsanitizer` 防恶意 JS / 嵌入对象

## 八、监控指标

| 指标 | 告警阈值 |
| :--- | :--- |
| 上传失败率 | > 1% (5min 滑动) |
| 炸开任务平均时长 | > 60s/MB |
| Celery 队列长度 | > 1000 |
| S3 5xx 率 | > 0.1% |
| 病毒命中数 | > 0 立即告警 |
