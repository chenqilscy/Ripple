# G3 · 冰山 (Iceberg) 资产沉淀模块设计

**版本：** v1.0
**日期：** 2026‑04‑21
**适用对象：** 后端 / 前端 / 产品
**地位：** 「青萍」**出口模块**——固态资产的快照、可视化、导出与复用。

---

## 一、设计哲学

> "人们只看到海面上壮丽的冰山一角，却忽略了海面下庞大的支撑。"

冰山解决三件事：
1. **沉淀**：将活水（造浪池）冻结为不可变的快照（Snapshot）
2. **展示**：以"灵感三角洲"图揭示创作的演化路径
3. **流通**：导出 / 公开分享 / 跨项目复用（决堤口的三种出口）

---

## 二、核心概念

### 2.1 三层数据结构

```
Lake (活水)
  │
  ▼ [快照]
Snapshot (冻结的图，含节点与连边的不可变副本)
  │
  ▼ [策展]
Iceberg (冰山，对一个或多个快照的策展视图，可发布)
  │
  ▼ [发布出口]
├─ Export      (PDF / Markdown / HTML / 视频)
├─ Share Link  (公开链接，朦胧/明文)
└─ Tide Square (社区潮汐广场)
```

### 2.2 与造浪池的关系

| 概念 | 状态 | 可改 | 谁可见 |
| :--- | :--- | :--- | :--- |
| Lake | 活水 | ✅ | 湖泊成员 |
| Snapshot | 冻结 | ❌ | 仅冰山所有者 |
| Iceberg | 策展 | 元数据可改 | 按 visibility |

## 三、Snapshot 数据结构

延伸 [G1 §2.1](./数据模型与权限设计.md) 的 `:Snapshot` 标签：

```cypher
(:Snapshot {
  id, iceberg_id, taken_at, node_count, edge_count,
  source_lake_id,
  // 内容指纹
  content_hash: STRING,    // SHA-256(序列化所有节点与边)
  storage_uri: STRING      // 大快照可外放对象存储
})

(:Snapshot)-[:CONTAINS]->(:Node)
(:Snapshot)-[:FROZEN_EDGE {type, weight}]->(:Snapshot)  // 边也快照
```

**冻结策略：**
- 小快照（< 500 节点）：直接在 Neo4j 中保留 `[:CONTAINS]` 关系
- 大快照（≥ 500 节点）：序列化为 JSON 落 S3，Neo4j 仅留元数据

## 四、灵感三角洲视图

### 4.1 算法定义

输入：一个 Snapshot
输出：一张**有向无环的演化拓扑图**

**节点分层（Layout）**：
1. **源头层 (Source)**：无入向 RELATES_TO 的节点
2. **支流层 (Tributary)**：有少量入向边的中间节点
3. **汇流层 (Confluence)**：高入度节点（关键转折点）
4. **冰山层 (Tip)**：被标记为"成品"的节点

**视觉设计：**
- 时间从下往上（湖底→海面）
- 节点大小 ∝ 出入度
- 暗流颜色编码：用户创建（深紫）vs AI 推断（淡紫）
- 关键路径高亮："决堤口"被点击时，反向追溯所有贡献节点的最大权重路径

### 4.2 计算 API

```http
GET /api/v1/icebergs/{id}/delta
  → 返回 GeoJSON 风格的拓扑数据，前端用 WebGL 渲染
```

```json
{
  "nodes": [{ "id", "content", "layer", "in_degree", "out_degree", "is_tip" }],
  "edges": [{ "source", "target", "type", "weight", "by" }],
  "critical_path": ["uuid1", "uuid2", "uuid3"]
}
```

## 五、决堤口（发布）

### 5.1 三种出口

| 出口 | 入口 | 接收方 | 可逆性 |
| :--- | :--- | :--- | :--- |
| **入海 (Export)** | `POST /icebergs/{id}/export` | 文件下载 | 完全本地，可反复 |
| **抛饵 (Share)** | `POST /icebergs/{id}/share-links` | 任何人凭链接 | 可撤销链接 |
| **入流 (Publish)** | `POST /icebergs/{id}/publish` | 青萍社区 | 可下架 |

### 5.2 导出格式

| 格式 | 用途 | 实现 |
| :--- | :--- | :--- |
| **Markdown** | 文章 / 知识库 | Pandoc-like 模板，按关键路径线性化 |
| **PDF** | 答辩 / 汇报 | Headless Chrome 渲染三角洲 + 文字稿 |
| **HTML (单文件)** | 静态分享 | 内嵌 SVG + JS，可独立打开 |
| **视频 (MP4)** | 社交传播 | 后端 ffmpeg + 时间轴动画（节点按时间逐个浮现） |
| **JSON** | 第三方接入 | 完整 Snapshot 序列化 |

导出为**异步任务**，完成后通过邮件 / WebSocket 通知用户。

### 5.3 抛饵（公开分享）

```http
POST /api/v1/icebergs/{id}/share-links
{
  "visibility": "BLURRED",         // 朦胧版 (推荐) | CLEAR
  "password": "optional",
  "expires_in_days": 7,
  "allow_bottles": true,
  "anti_screenshot_watermark": true
}
```

**朦胧版渲染规则：**
- 节点形状、颜色、连边可见
- 节点内容**模糊化**为占位文字（"……"或字符级哈希）
- 强制叠加**动态水印**（IP + UA + 时间戳）
- 前端检测截屏 API（部分平台）则发出告警

### 5.4 漂流瓶回流

详见 [G1 §四 `bottles` 表](./数据模型与权限设计.md)。

防滥用：单一 `sender_fingerprint` 24h 内最多 5 瓶/冰山，超出转入审核队列。

## 六、冰山复用（场景 10）

```http
POST /api/v1/lakes/{target_lake_id}/import-from-iceberg
{
  "iceberg_id": "<source>",
  "node_ids": ["uuid", "uuid", ...],   // 选择性导入
  "mode": "REFERENCE | CLONE"
}
```

### 6.1 REFERENCE 模式
- 在 `target_lake` 中以 `[:REFERENCES]` 关系链接源节点
- 半透明 + ⛵ 显示，只读
- 源冰山被 ERASED 时，触发 `GHOST_OF` 转换（见 G1 §三）

### 6.2 CLONE 模式
- 节点完整拷贝到目标湖（深复制）
- 切断与源冰山的关联，独立演化
- 元数据保留 `cloned_from_iceberg_id`（用于溯源审计）

## 七、权限与可见性

| 操作 | 所需角色 |
| :--- | :--- |
| 创建快照 / 冰山 | 源湖 OWNER 或 NAVIGATOR |
| 删除冰山 | 仅 OWNER |
| 创建 Share Link | OWNER |
| 创建 Export | OWNER / NAVIGATOR / PASSENGER（仅自己创建的冰山） |
| 复用引用 | 源冰山 visibility ≥ TEAM 且当前用户在源团队 / 或 PUBLIC |
