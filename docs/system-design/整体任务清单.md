> 优先级评估基准：用户价值 × 技术可行性 ÷ 工作量

### P12-A：批量节点导入（CSV / JSON） · ✅ 已落地（含 1000 行验收）

| 属性 | 说明 |
|------|------|
| 目标 | 允许用户通过上传 CSV/JSON 文件一次性创建多个节点（含标签、内容、边关系） |
| 后端 | ✅ `POST /api/v1/lakes/{id}/nodes/batch`（`handlers_node.go::BatchImport` + `service/node.go::BatchImportNodes`，`maxBatchImport=1000`，超出返回 `ErrInvalidInput`） |
| 前端 | ✅ `frontend/src/components/ImportModal.tsx` + Home 页 📥 按钮 |
| 测试 | ✅ `internal/service/node_batch_import_test.go`：`TestNode_BatchImport_1000Rows` / `_BadRowsSkipped` / `_OverLimit` 全部 PASS |
| 验收 | 1000 行成功率 100%；空/纯空白行跳过；超过 10000 字符自动截断；> 1000 行返回 400 |

### P12-B：GraphQL API 层 · ⏸ 暂缓

| 属性 | 说明 |
|------|------|
| 目标 | 在现有 REST 之上提供 GraphQL 端点，减少前端多次 fetch |
| 技术 | `github.com/99designs/gqlgen`（Go codegen） |
| 范围 | 只读查询（lakes / nodes / edges / members）+ 订阅（节点变更推送） |
| 估算 | XL（约 4 周） |
| 决策 | 当前 REST 性能足够，前端聚合需求不强；优先完成 P12-E 与 TD-04 |

### P12-C：多租户 / 组织（Organization Scoping） · ✅ 已落地（基础形态）

| 属性 | 说明 |

| 路由 | ✅ `/api/v1/organizations`（CRUD + 成员管理）+ `PATCH /api/v1/lakes/{id}/org` 绑定湖归属 |
| 前端 | ✅ `frontend/src/components/OrgPanel.tsx` |
| 缺口 | Org 级邀请流（vs 当前的 Lake 级邀请）尚未拉通；Org 配额/计费策略未定义（不在 P12 范围） |

### P12-D：全文检索 · ✅ 已落地（实现偏离原计划）

| 属性 | 说明 |
|------|------|
| 目标 | 节点内容全文检索，支持中英文分词 |
| 实际技术 | ⚠️ **Neo4j fulltext index `node_content_fts`**（`backend-go/scripts/neo4j_constraints.cypher` 声明），非原计划的 PG `tsvector` |
| 后端 | ✅ `GET /api/v1/search?q=...&lake_id=...&limit=...`（`handlers_node.go::Search` + `service/node.go::SearchNodes` + `repo_node.go::cypherSearchNodes`） |
| 前端 | ✅ `SearchModal.tsx` + Home 页 Cmd/Ctrl+K 快捷键 |
| 决策记录 | 选择 Neo4j 而非 PG tsvector 的原因：节点主存于 Neo4j，避免双写一致性；`zhparser`/`pg_jieba` 在生产部署需扩展安装 |
| 后续 | 在 10k 节点数据下采集 p95 数据并补入 `M3-性能基线对照.md`（候选） |
