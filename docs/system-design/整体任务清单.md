## Phase 12 候选任务

> 优先级评估基准：用户价值 × 技术可行性 ÷ 工作量
> 接受本阶段任务前需经十角色团队流程评审。

### P12-A：批量节点导入（CSV / JSON）

| 属性 | 说明 |
|------|------|
| 目标 | 允许用户通过上传 CSV/JSON 文件一次性创建多个节点（含标签、内容、边关系） |
| 后端 | `POST /api/v1/lakes/{id}/nodes/batch` · PG/Neo4j 事务批量写入 |
| 前端 | 文件拖拽上传 → 字段映射预览 → 确认导入 |
| 价值 | 降低数据迁移成本；适合将现有知识库接入 Ripple |
| 估算 | L（约 2 周） |

### P12-B：GraphQL API 层

| 属性 | 说明 |
|------|------|
| 目标 | 在现有 REST 之上提供 GraphQL 端点，减少前端多次 fetch |
| 技术 | `github.com/99designs/gqlgen`（Go codegen） |
| 范围 | 只读查询（lakes / nodes / edges / members）+ 订阅（节点变更推送） |
| 价值 | 外部开发者友好；前端可按需取字段 |
| 估算 | XL（约 4 周） |
| 风险 | 需与接口设计师评审，避免破坏现有 REST 合约 |

### P12-C：多租户 / 组织（Organization Scoping）

| 属性 | 说明 |
|------|------|
| 目标 | 在 Spaces 之上增加 Organization 层，支持企业多团队隔离 |
| 范围 | Org 创建/邀请/角色 + Lake/Space 归属 Org + 数据隔离 |
| 价值 | SaaS 商业化的核心能力 |
| 估算 | XL（约 6 周） |
| 风险 | 破坏现有数据模型；需完整迁移脚本 |

### P12-D：全文检索（PG tsvector）

| 属性 | 说明 |
|------|------|
| 目标 | 节点内容全文检索，支持中英文分词 |
| 技术 | PG `tsvector` + `GIN` 索引；中文分词用 `zhparser`（或 `pg_jieba`） |
| 后端 | `GET /api/v1/search?q=...&lake_id=...` |
| 前端 | 全局搜索框（Cmd+K） |
| 价值 | 知识图谱的核心访问路径 |
| 估算 | M（约 1.5 周） |

### P12-E：移动端 PWA 完善

| 属性 | 说明 |
|------|------|
| 目标 | 补全 M4-C：离线写节点合并（IndexedDB + Yjs） + 推送通知 |
| 技术 | Workbox 离线缓存策略 + Web Push API + VAPID |
| 价值 | 随时随地记录想法；与知识图谱自动同步 |
| 估算 | L（约 2.5 周） |

## Phase 12 优先级建议

| 优先级 | 任务 | 理由 |
|--------|------|------|
| P1 | **P12-D** 全文检索 | 工作量小、用户价值高，直接改善核心使用路径 |
| P2 | **P12-A** 批量导入 | 降低迁移门槛，加速用户增长 |
| P3 | **P12-E** PWA 完善 | 补全 M4-C 承诺，提升移动体验 |
| P4 | **P12-C** 多租户 | 商业化必要，但工作量大，暂排期 |
| P5 | **P12-B** GraphQL | 技术债不紧迫，先验证产品价值再引入 |
