# G2 · 云霓 (Cloud) 灵感采集模块设计

**版本：** v1.0
**日期：** 2026‑04‑21
**适用对象：** 移动端 / 后端 / 产品
**地位：** 「青萍」**入口模块**——气态灵感的捕捉、暂存与凝露入水。

---

## 一、设计哲学

> "灵感不应被强行捕获，而应像云霓般自在。"

云霓是产品的"**漏斗顶部**"，要求**比微信收藏更快、比备忘录更轻、比脑机更安静**：
1. **零摩擦**：从想法产生到入库 ≤ 3 秒
2. **零打扰**：不弹通知、不索取归类、不要求标题
3. **可后悔**：所有捕获均为雾态，未"凝露"前不污染创作画布

## 二、采集形态（输入通道）

| 通道 | 触发方式 | 内容形态 | 资产存储 |
| :--- | :--- | :--- | :--- |
| **桌面长按图标** | 长按 App 图标 / 桌面小组件 | 文本、语音 | 本地→云端 |
| **快捷键** | 全局 `Ctrl+Shift+Q` | 文本 | 直传 |
| **截图捕梦** | 通知栏长按截图 | 图像 + OCR 文本 + 涂鸦 | Asset(IMAGE) |
| **语音呢喃** | 长按麦克风按钮 | 语音 + ASR 转写 | Asset(VOICE) + content |
| **分享面板** | iOS/Android 系统分享 → 青萍 | URL/文本/图 | 视类型 |
| **API 接入** | 第三方（剪藏插件、IoT） | 任意 | 由调用方决定 |

## 三、雾气节点状态机（与 G1 对齐）

```
[采集动作]
    │
    ▼
[本地队列] ──网络可用──► [上传 API] ──► (:Node {state:'MIST', ttl_at:+7d})
    │                                          │
    │ 离线                                     │
    ▼                                  [展示在云霓 InBox]
[本地 SQLite/IndexedDB]                      │
                                             ├─ 用户主动「凝露」 ─► state=DROP, ttl_at=NULL
                                             ├─ 用户主动「蒸发」 ─► state=VAPOR
                                             └─ 7 天到期未处理 ──► 自动 VAPOR + 邮件提醒
```

## 四、客户端架构

### 4.1 移动端（React Native / Flutter 二选一，推荐 RN）

```
┌────────────────────────────────────────┐
│  Quick Capture UI（桌面图标长按）       │ ← 0.5s 内可输入是 KPI
├────────────────────────────────────────┤
│  Capture Service（前台/后台）           │
│  ├─ ASR Engine (本地 Whisper-tiny)      │
│  ├─ OCR Engine (本地 ML Kit)            │
│  └─ Compress (图片 < 1MB)               │
├────────────────────────────────────────┤
│  Outbox Queue (SQLite)                  │
│  - 容量：≥ 500 条                       │
│  - 溢出策略：FIFO 驱逐已凝露项          │
├────────────────────────────────────────┤
│  Sync Worker (WorkManager / BGTask)     │
│  - 指数退避：1s, 5s, 30s, 5m, 1h        │
│  - 网络可用 + 充电时优先批量同步        │
└────────────────────────────────────────┘
```

- 系统托盘常驻"待凝露区"小红点
- 与移动端共享同一后端 InBox

## 五、API 设计

### 5.1 上传雾气节点

```http
POST /api/v1/cloud/mist
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "client_id": "uuid-v4 (幂等键，本地生成)",
  "captured_at": "2026-04-21T08:00:00Z",
  "channel": "VOICE | TEXT | IMAGE | URL",
  "content": "文本内容或简短描述",
  "asset_upload_token": "<可选，预签名 URL token>",
  "context": {
    "device": "iPhone 15",
    "location": "optional, 用户授权",
    "lake_hint": "<可选，目标湖 ID>"
  }
}
```

**响应：**
```json
{
  "node_id": "uuid",
  "state": "MIST",
  "ttl_at": "2026-04-28T08:00:00Z",
  "inbox_url": "https://app.qingping.io/cloud/inbox"
}
```

**幂等性：** 服务端按 `(user_id, client_id)` 去重，重复请求返回原 `node_id`。

### 5.2 凝露入水

```http
POST /api/v1/cloud/condense
{
  "node_ids": ["uuid", "uuid"],
  "target_lake_id": "uuid",
  "position_hint": "AUTO | CENTER | { x, y }"
}
```

**服务端处理：**
1. 验证用户对 `target_lake_id` 至少 PASSENGER 权限
2. 节点 `state: MIST → DROP`，清空 `ttl_at`
3. 异步触发：① 生成 embedding ② AI 探索潜在暗流（详见 G5）
4. 通过 WebSocket 推送给湖泊在线成员

### 5.3 蒸发雾气节点

```http
POST /api/v1/cloud/vaporize
{ "node_ids": ["uuid"] }
```

进入 VAPOR 状态，30 天迷雾区可恢复（详见 G1）。

### 5.4 列出 InBox

```http
GET /api/v1/cloud/inbox?cursor=<id>&limit=50&channel=ALL
```

游标分页，按 `captured_at` 倒序。

## 六、TTL 与生命周期 Job

### 6.1 后台 Job（每日运行）

```python
# pseudocode
@scheduled(cron="0 3 * * *")
def expire_mist_nodes():
    # 1. 找出 7 天 TTL 已过期且仍为 MIST 的节点
    expired = neo4j.run("""
        MATCH (n:Node {state: 'MIST'})
        WHERE n.ttl_at < datetime()
        RETURN n.id, n.created_by
    """)

    for record in expired:
        # 2. 转为 VAPOR
        transition_to_vapor(record["n.id"])
        # 3. 提前一天发送邮件提醒（在 D-1 由另一 Job 完成）
```

### 6.2 D-1 提醒 Job（在过期前 24h）

发送邮件："**您有 N 缕云霓即将蒸发，是否凝露入水？**" 附 InBox 直达链接。

## 七、与造浪池的边界

| 关注点 | 云霓 | 造浪池 |
| :--- | :--- | :--- |
| 节点状态 | MIST | DROP / FROZEN |
| 数据可见性 | 仅创建者 | 按湖泊权限 |
| AI 行为 | **不**自动建暗流（节省成本） | 全力 |
| 视觉表现 | 雾气、半透明、漂浮在湖面之上 | 浮萍、贴近湖面 |
| 入口 | 移动端为主 + 桌面快捷 | Web 主战场 |

> **关键决定**：MIST 节点**不**进入向量检索（避免噪音），凝露后才生成 embedding。

| 指标 | SLO |

## 九、安全与隐私

1. 语音/图片资产传输：HTTPS + 端到端 AES-GCM（密钥派生自用户主密钥）
2. 本地 SQLite 加密：SQLCipher，密钥存 OS Keychain
3. 位置信息：默认关，用户显式开启
4. 录音权限：仅在长按麦克风时申请，不后台录音
5. 截图 OCR：默认在端侧完成，不上传原图除非用户开启"高质量识别"
