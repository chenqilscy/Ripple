## 一、设计理念

### 1.1 核心定义

**Delta View（三角洲视图）** 是「造浪池」的核心交互形态。它打破传统软件"菜单栏 + 画布"的工业格局，将整个屏幕重塑为一片**无边界的有机水域**。

### 1.2 设计原则

- **去工具化 (De-tooling)**：减少永久性 UI 控件，让内容（灵感节点）成为绝对主角
- **流体隐喻 (Fluid Metaphor)**：界面元素具备液体的物理特性（流动、融合、表面张力）
- **呼吸感 (Negative Space)**：大量留白，模拟水面的空旷与宁静

## 二、界面布局：无边界画布

### 2.1 顶部悬浮栏

- **形态：** 极窄半透明毛玻璃长条，悬浮顶部，不占用画布空间
- **左侧：** Logo（青萍）、全局返回
- **中部：** 当前项目名称（可编辑，默认"未命名的水域"）
- **右侧：** 三个全局行动点
  - `[☰]` 全局菜单。**MVP 必含 4 项：** ① 我的湖泊  ② **迷雾区**（VAPOR 节点恢复 / 30 天倒计时）  ③ 设置  ④ 帮助 / 教程重看
  - `[🌊]` **固形** (Save)
  - `[🚀]` **决堤口** (Publish)

### 2.2 中央水域（核心区）

- **占屏：** ≥ 90% 的动态流体背景
- **背景：** 带微弱噪波 (Noise) 与 Ripple (Wave) 的动态纹理；色彩随昼夜模式或用户设定变化（默认青碧渐变）
- **交互：** 无限缩放、惯性拖拽（手指划过后水面继续滑动并逐渐停止）

### 2.3 底部状态指示器

- 极细状态条
- 显示网络密度、AI 工作状态（如"正在探测地下暗河..."）、降级提示（详见 [G5 §五](AI服务编排-织网工作流.md)）

## 三、核心元素规范

### 3.1 节点（浮萍）

| 属性 | 规范 |
| :--- | :--- |
| 形状 | **有机圆形 (Organic Blob)**，边缘有荷叶般的不规则起伏 |
| 材质 | **磨砂玻璃 (Glassmorphism)**：背景模糊，前景高亮，边缘内发光 |
| 色彩 | 类型色（文学：青碧；科技：深蓝），透明度 70% |
| Hover | 微放大 (Scale 1.05)，产生一圈微弱涟漪 |
| Active | 边缘高亮，水珠被选中的紧绷感 |

详细状态机：见 [D2 节点与连线状态机](节点与连线状态机规范.md)。

### 3.2 连线（暗流）

| 属性 | 规范 |
| :--- | :--- |
| 路径 | 贝塞尔曲线，自然垂坠和弧度 |
| 动效 | 粒子流：细小光点沿曲线从源向目标流动；速度 ∝ 关联强度 |
| 样式 | 强关联实线，AI 推测弱关联虚线 |

## 四、交互逻辑：隐形操控杆

### 4.1 上下文浮现

- 长按空白处 → 浮现"投石"菜单（输入灵感）
- 选中节点 → 节点周围浮现"生长"按钮（分蘖、探源、蒸发）

### 4.2 手势定义

| 动作 | 手势 | 视觉反馈 |
| :--- | :--- | :--- |
| 投石 (Create) | 双击空白 | 涟漪 → 生成节点 |
| 引流 (Link) | 拖拽出水流连线 | 松手建立水道 |
| 分蘖 (Branch) | 拖拽节点向外拉 | 像拉面条拉出新分支 |
| 固形 (Save) | 点击节点 | 冰晶凝结动画，边框变实 |
| 蒸发 (Delete) | 节点向上飘散 | 软删，迷雾区可恢复 |

### 4.3 视图模式切换

- **俯瞰模式 (Delta View)**：全局视角；节点像荷叶漂浮，连线流动曲线带粒子
- **聚焦模式 (Deep Dive)**：选中节点 → 右侧浮现"生长引擎"，显示该节点的根系与藤蔓建议

## 五、技术实现建议

### 5.1 渲染

- **渲染引擎：** Three.js / PixiJS（WebGL）
- **图布局：** D3-force（仅计算位置，渲染交给 WebGL）
- **状态管理：** Zustand

### 5.2 关键 CSS 提示

```css
.node {
  backdrop-filter: blur(10px);             /* 毛玻璃 */
  border-radius: 50%;                       /* 圆形 */
  box-shadow: inset 0 0 10px rgba(255,255,255,0.5);  /* 内发光 */
}
```

详细 Shader 实现：见 [D3 流体动效引擎](流体动效引擎-WebGL.md)。
