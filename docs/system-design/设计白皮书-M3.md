# M3 设计白皮书 · 凝结与共生

> 阶段定位：从「发散+协作」走向「凝结+反哺+多空间」  
> 时间窗：2025-Q4 末 ~ 2026-Q1  
> 上游依赖：M2 已交付的造浪池、邀请协作、节点编辑历史、在线状态、6 家 LLM Provider

---

## 1. M3 三大主题

### 1.1 凝结（Crystallize）— 从云到水晶
M2 的造云已能把发散文字变成 MIST 节点；M3 进一步：
- **凝露 → 凝结**：把湖里多个相关 DROP/CLOUD 节点凝结成一个 **PERMA 节点**（永久节点，不会蒸发）
- PERMA 节点支持：
  - 多模态（文/图/音/视频引用）
  - 长文本（≥ 5K 字）
  - LLM 摘要 + 关键词 + 多语言翻译
- 凝结操作：用户选定 N 个节点 + 选 PERMA 模板 → 后台异步任务生成 → 通知

### 1.2 反哺（Feedback）— AI 学习用户
- 每次用户对 AI 输出做的「采纳/修改/丢弃」都作为 implicit signal 落 `feedback_events` 表
- 每周离线任务汇总 → 生成「用户偏好画像」（关键词、风格、负面词）
- 后续造云时把画像注入 system prompt（个人化发散）
- **不做训练**：仅 prompt-time personalization；保护隐私

### 1.3 多空间（MultiSpace）— 工作区分层
- 用户可创建多个 **空间（Space）**，每个空间下有多个湖
- 空间级权限：成员、共享 LLM 配额、共享 PERMA 节点池
- 空间间不互通；切换空间清空 active lake
- 用于区分：个人创作 / 团队协作 / 客户项目

## 2. 数据模型扩展

### 2.1 新表

```sql
-- spaces：工作空间
CREATE TABLE spaces (
  id UUID PRIMARY KEY,
  owner_id UUID NOT NULL REFERENCES users(id),
  name TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  llm_quota_monthly INT NOT NULL DEFAULT 10000,  -- token quota
  llm_used_current_month INT NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_spaces_owner ON spaces(owner_id);

-- space_members：空间成员
CREATE TABLE space_members (
  space_id UUID NOT NULL REFERENCES spaces(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('OWNER','EDITOR','VIEWER')),
  joined_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (space_id, user_id)
);

-- lakes 增字段：space_id（NULL=个人湖；与 owner_id 二选一关联）
ALTER TABLE lakes ADD COLUMN space_id UUID REFERENCES spaces(id) ON DELETE SET NULL;

-- nodes 扩展：state 加 PERMA
ALTER TABLE node_states_check  -- 实际是 Cypher 端，迁移用应用层校验
  -- :Node.state ∈ {DROP,MIST,CLOUD,VAPOR,PERMA}

-- perma_nodes：PERMA 节点元数据（重数据结构，单独存）
CREATE TABLE perma_nodes (
  node_id TEXT PRIMARY KEY,           -- = nodes.id（Neo4j）
  template TEXT NOT NULL,              -- 'summary' / 'mindmap' / 'narrative' / 'qa'
  source_node_ids TEXT[] NOT NULL,    -- 凝结自哪些节点
  long_content TEXT NOT NULL,          -- 主体长文（>5K 允许）
  summary TEXT NOT NULL DEFAULT '',
  keywords TEXT[] NOT NULL DEFAULT '{}',
  translations JSONB NOT NULL DEFAULT '{}'::jsonb,  -- {en:"...", ja:"..."}
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- feedback_events：用户对 AI 输出的反馈
CREATE TABLE feedback_events (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id),
  source_node_id TEXT NOT NULL,        -- AI 产出的节点
  action TEXT NOT NULL CHECK (action IN ('ADOPT','EDIT','DISCARD','EVAPORATE')),
  payload JSONB NOT NULL DEFAULT '{}'::jsonb,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_feedback_user_time ON feedback_events(user_id, created_at DESC);

-- user_preferences：周聚合后的画像（去隐私化）
CREATE TABLE user_preferences (
  user_id UUID PRIMARY KEY REFERENCES users(id),
  positive_keywords TEXT[] NOT NULL DEFAULT '{}',
  negative_keywords TEXT[] NOT NULL DEFAULT '{}',
  preferred_tone TEXT NOT NULL DEFAULT '',  -- 'analytical' / 'poetic' / ...
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 2.2 Neo4j 扩展

```cypher
// PERMA 节点：和 DROP/MIST/CLOUD 一样是 :Node，state='PERMA'，多一个 :Perma 标签
CREATE CONSTRAINT perma_node_id IF NOT EXISTS
FOR (p:Perma) REQUIRE p.id IS UNIQUE;

// 凝结边：(:Perma)-[:CRYSTALLIZED_FROM]->(:Node)
// 用于追溯，删 PERMA 时不级联删 source
```

## 3. 关键 API

### 3.1 凝结
```
POST /api/v1/perma/crystallize
  { lake_id, source_node_ids: [...], template: 'summary'|'mindmap'|'narrative'|'qa' }
  → { task_id }   // 异步，订阅 WS 'perma.created'

GET /api/v1/perma/{node_id}
  → { ...PermaNode }

GET /api/v1/spaces/{id}/perma   // 空间级 PERMA 池
  → { perma_nodes: [...] }
```

### 3.2 反哺
```
POST /api/v1/feedback
  { source_node_id, action, payload }
  → 204
// 隐式触发：Adopt 即把节点 link 到 PERMA；Edit 即记 diff；Discard 即标弃；Evaporate 系统自动产生
```

### 3.3 空间
```
POST /api/v1/spaces                   { name, description }
GET  /api/v1/spaces                   → { spaces: [...] }   // 我所属
GET  /api/v1/spaces/{id}              → Space + members + quota
POST /api/v1/spaces/{id}/members      { email, role }
DELETE /api/v1/spaces/{id}/members/{user_id}
PUT  /api/v1/lakes/{id}/space         { space_id }   // 把湖移入空间
```

## 4. 架构变化

### 4.1 新 worker
- **CrystallizeWorker**：消费 `crystallize_tasks` 队列，调 LLM 生成 long_content/summary/keywords/translations，落 perma_nodes + 创建 :Perma Neo4j 节点
- **PreferenceAggregator**（cron 周一凌晨）：扫 `feedback_events` 最近 7 天 → 更新 `user_preferences`

### 4.2 LLM Router 改造
- 新增 `ModalityLong`（区别 ModalityText，max_tokens 默认 4096）
- 必须支持流式（参考 [技术债清单.md](技术债清单.md) TD-001）
- AIWeaver / CrystallizeWorker 在 system prompt 注入用户 `preferred_tone` + `positive_keywords`

### 4.3 配额系统
- Space 级 LLM token quota：每月 reset
- 每次 LLM 调用前在 `spaces.llm_used_current_month` 上 atomic INC + 校验
- 超额拒绝 + WS 通知 OWNER

## 6. 风险与权衡

| 风险 | 缓解 |
|------|------|
| PERMA long_content 5K 字 → 数据库膨胀 | PG TOAST 压缩 + 单 Space 限 1000 PERMA（M3 内） |
| 用户偏好画像隐私 | 不存原文，仅存关键词；用户可一键清除 |
| Space + Lake 双层模型增加心智负担 | 默认有「我的空间」，新湖默认入此空间，老用户无感 |
| 凝结调 LLM 长 prompt → 成本飙升 | 必须用流式 + 上限 quota；本地 Ollama 兜底 |

## 9. 知识图谱语义增强（M3 设计评审补充 · 2026-04-28）

> 触发：M3 设计评审，核心议题之一。

### 9.1 目标
为每个节点生成语义 embedding，支持语义相似检索（"找和这个节点概念相近的节点"），为 M4 RAG 流水线打基础。

### 9.2 基础设施
- **当前状态**：本节保留为 M3 设计评审备选方案；pgvector / `node_embeddings` 支线已从当前主线工作区回收，现行 compose 仍使用 `postgres:16-alpine`
- **新表**：
  ```sql
  CREATE EXTENSION IF NOT EXISTS vector;

  CREATE TABLE node_embeddings (
    node_id    TEXT        NOT NULL,
    lake_id    TEXT        NOT NULL,
    model      TEXT        NOT NULL DEFAULT 'zhipu-embedding-3',
    vector     vector(2048),          -- 维度依 model 决定，pgvector 支持 ≤ 16000
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (node_id, model)
  );
  CREATE INDEX ON node_embeddings USING ivfflat (vector vector_cosine_ops) WITH (lists = 100);
  ```
- 存 `model` 字段：切换模型后旧向量仍存在，查询时按 model 过滤；重新 embed 任务只重建指定 model

### 9.3 异步 Embedding Worker
- 节点创建/内容更新 → Outbox 写入 `embedding_tasks` 类型事件
- `EmbeddingWorker`（独立 goroutine pool，3 worker）消费，调 LLM embedding API，写入 `node_embeddings`
- 失败重试 3 次，超过则打 ERROR 日志（不阻塞主流程）

### 9.4 语义搜索 API
```
POST /api/v1/lakes/{lake_id}/nodes/semantic-search
  Body: { query: string, limit?: int(default 10), threshold?: float(default 0.7), model?: string }
  → { nodes: [{ ...NodeItem, score: float }] }
```
- 先生成 query 的 embedding，再用 `<=>` 余弦距离排序
- `threshold` 过滤相似度 < 阈值的结果
- 权限：lake 可读即可访问

## 10. AI Weaver 图感知进化（M3 设计评审补充 · 2026-04-28）

> 触发：M3 设计评审，核心议题之一。

### 10.1 现状
AIWeaver 造云时 prompt 仅包含 `{node.content}`，不感知图拓扑。

### 10.2 邻居上下文注入
造云时，在 prompt 中追加目标节点的一跳邻居节点内容（Neo4j 查询）：

```cypher
MATCH (n:Node {id: $nodeId})-[:LINKED_TO|LINKED_FROM]-(neighbor:Node)
WHERE neighbor.state IN ['DROP','PERMA']
RETURN neighbor.id, neighbor.content
LIMIT 5
```

- 每邻居 content 截前 200 字
- 最多 5 个邻居
- 总追加不超过 4000 字符（约 1K token）防止超限

Prompt 结构变化：
```
你是一个发散助手。下面是节点内容：
{node.content}

（可选）关联节点（仅供参考，勿直接复制）：
- {neighbor1_content_truncated}
- {neighbor2_content_truncated}
...

请基于以上内容，发散出 {count} 个新想法节点。
```

### 10.3 实现位置
`internal/service/ai_weaver.go` 中 `buildPrompt(node)` 函数，调 Neo4j 查邻居。
- 若 Neo4j 查询失败，静默降级（不带邻居上下文），不影响主流程。

## 11. 多 Lake 跨图查询（M3 设计评审补充 · 2026-04-28）

> 触发：M3 设计评审，核心议题之一。

### 11.1 目标
同一 Space 下多个 lake 孤岛化，AI 无法综合分析。跨湖查询允许用户在一次请求中搜索多个 lake。

### 11.2 API 设计
```
POST /api/v1/spaces/{space_id}/cross-query
  Body: {
    query: string,
    lake_ids?: [string],  // 不传 = 查 Space 内所有可读 lake
    limit?: int,          // per-lake limit，default 5
    model?: string
  }
  → {
    results: [{
      lake_id: string,
      lake_name: string,
      nodes: [{ ...NodeItem, score: float }]
    }]
  }
```

### 11.3 实现
- 鉴权：只查调用者有读权限的 lake（复用 `LakeService.canRead()`）
- 并发扇出：每个 lake 一个 goroutine，WaitGroup + channel 归并
- 超时：5s 强制返回已收到的结果（context.WithTimeout）
- 结果按 lake 分组，每 lake 最多 limit 条，按 score 降序

### 11.4 依赖
- §9 语义搜索（cross-query 底层调语义搜索）
- §1.3 多空间（Space 模型）
- 里程碑：M3-S5 之后，作为 M3-S6 新增 Sprint
