## 一、节点交互状态机（前端层）

节点存在四种核心**交互态**：`Default`（常态）、`Hover`（悬停）、`Active`（激活/选中）、`Disabled`（蒸发/禁用）。

> ⚠️ 注意：交互态与 [G1 数据态](G1-数据模型与权限设计.md) 的映射：DROP→{Default/Hover/Active}；VAPOR→Disabled；FROZEN→Active 子态（已固形高亮）。

### 1.1 状态流转

```
       [Mouse Enter]
Default ─────────► Hover
   ▲ │              │ ▲
   │ │ [Click]      │ │ [Mouse Leave]
   │ ▼              │ │
   Active ◄─────────┘ │
   ▲                  │
   │ [Context Action: 蒸发]
   ▼
   Disabled
```

### 1.2 状态属性表

| 状态 | Default | Hover | Active | Disabled |
| :--- | :--- | :--- | :--- | :--- |
| 尺寸 | 1.0× | 1.08× | 1.1× | 0.5× |
| 背景色 | `rgba(46,139,87,0.6)` | `rgba(46,139,87,0.75)` | `rgba(46,139,87,0.9)` | `rgba(200,200,200,0.3)` |
| 边框 | None | 1px solid white | 2px solid #FFFFFF | dashed |
| 阴影 | `0 4px 15px rgba(0,0,0,0.1)` | `0 8px 25px rgba(46,139,87,0.3)` | `0 0 20px rgba(255,127,80,0.5)` | None |
| 文字 | #FFFFFF | #FFFFFF | #FFFFFF | #AAAAAA |
| 动效 | 轻微浮动 | 涟漪扩散 | 脉冲呼吸 | 向上飘散 |

### 1.3 关键动效参数

```css
@keyframes ripple {
  from { transform: scale(1); opacity: 1; }
  to   { transform: scale(1.5); opacity: 0; }
}

@keyframes pulse {
  0%, 100% { box-shadow: 0 0 10px rgba(255,127,80,0.5); }
  50%      { box-shadow: 0 0 25px rgba(255,127,80,0.8); }
}
```

## 二、连线（暗流）状态机

连线存在三种核心状态：`Potential`（潜在/虚连）、`Active`（激活/实连）、`Inactive`（非活跃）。

| 属性 | Potential | Active | Inactive |
| :--- | :--- | :--- | :--- |
| 线型 | 点划线 | 实线 | 虚线 |
| 颜色 | `rgba(106,90,205,0.4)` | `rgba(106,90,205,1)` | `rgba(150,150,150,0.2)` |
| 粗细 | 1.5px | 3px | 1px |
| 粒子流 | 慢速稀疏 (1× speed) | 快速密集 (2× speed) | 停止 |
| 箭头 | 无 | 末端小箭头 | 无 |

> Potential = AI 推断未确认 / 用户拖拽中临时连线；Active = 用户确认或 Critic 通过的关系；Inactive = 已被截流（断开）但保留历史。

### 2.1 暗流虚线交互（MVP M2 必做，体验方评审采纳）

AI 推断的 Potential 虚线必须**可被用户处理**，否则用户只能看着虚线发愁：

- **Hover 1s**：连线中点浮现两个胶囊按钮 `[✓ 接受]` `[✗ 否决]`
- **接受**：状态 Potential → Active，写入数据层 `[:RELATES_TO {confirmed: true}]`
- **否决**：连线淡出消失，写入 `ai_feedback` 表（用于评估 AI 接受率，见 [G6 §4.2](G6-可观测性与监控.md)）
- **忽略**：72h 内未操作的 Potential 连线自动降级为 Inactive（不删除，仅隐藏）

## 三、交互逻辑伪代码

```javascript
class Node {
  constructor(data) {
    this.state = 'DEFAULT';
    this.element = renderNode(data);
  }
  onMouseEnter() {
    if (this.state !== 'ACTIVE' && this.state !== 'DISABLED') {
      this.state = 'HOVER';
      this.element.animate({ scale: 1.08 }, { duration: 150 });
      this.playRippleEffect();
    }
  }
  onClick() {
    if (this.state === 'ACTIVE') openEditor(this);
    else {
      this.state = 'ACTIVE';
      this.element.animate({ scale: 1.1 }, { duration: 200 });
      this.startPulseAnimation();
    }
  }
  onEvaporate() {
    this.state = 'DISABLED';
    this.element.animate({ scale: 0.5, opacity: 0 },
      { duration: 500, onComplete: () => this.remove() });
    // 同时调用后端：state → VAPOR（详见 G1 §三）
  }
}

function startDragLink(sourceNode) {
  let tempLine = createTemporaryLine(sourceNode.position);
  onMouseMove((targetPos) => {
    tempLine.updateEndPoint(targetPos);
    tempLine.setStyle('dot-dash');  // Potential
  });
  onMouseUp((targetNode) => {
    if (targetNode && targetNode !== sourceNode) {
      createPermanentEdge(sourceNode, targetNode).setStyle('solid');  // Active
    } else {
      tempLine.fadeOutAndRemove();
    }
  });
}
```

## 四、开发者注意事项

1. **性能优化：** 节点 > 200 时强制使用 Canvas / WebGL（D3 SVG 模式会卡）
2. **贝塞尔曲线：** Tension ≈ 0.5 的 Cubic Bezier，控制点在节点切线方向上，避免连线交叉节点本体
3. **防抖：** Hover 触发涟漪需 50ms 延迟，避免鼠标快速划过触发大量动画
