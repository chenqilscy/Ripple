## 测试入口（中间件就绪场景）

```powershell
# 1) 基础连通性（本地或远端）
Test-NetConnection <host> -Port 15432
Test-NetConnection <host> -Port 16379
Test-NetConnection <host> -Port 7687

# 2) Go 集成最小链路（需 PG_URL）
$env:RIPPLE_INTEGRATION = '1'
$env:PG_URL = 'postgres://<user>:<pass>@<host>:15432/ripple?sslmode=disable'
go test ./internal/api/http/... -run "TestIntegrationHealthz|TestIntegrationAuthGuard" -count=1 -v
```

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 缺 `PG_URL` 导致集成测试 skip | 无法形成稳定回归门禁 | 统一在 `.env` + CI 注入并校验 |
| 搜索索引与写入路径耦合 | 写入延迟上升 | 先做异步更新策略评估 |
| 1000 WS 压测环境噪声大 | 指标波动不可比 | 固定压测窗口与机器规格 |
