## 一、两种连通模式

| 模式 | 隐喻 | 技术实质 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **运河（引用）** | 京杭大运河——船只通航，节点仍属各自湖 | 图中的 `[:REFERENCES]` 关系 | 跨湖只读引用 |
| **合流（合并）** | 江河入海——水体彻底融合 | Sub-graph 迁移 | 项目结束，归并入资产湖 |

## 二、Schema（详见 G1）

```cypher
[:REFERENCES]   起 Node → 止 Node    属性: { from_lake, permission, created_at }
[:MERGED_FROM]  起 Lake → 止 Lake    属性: { merged_at, by_user }
```

**重要约束：** `[:BELONGS_TO]` 关系唯一——引用节点**不**改变其所属湖。

完整定义：[G1 §2.2](G1-数据模型与权限设计.md)

## 三、运河模式（REFERENCES）实现

### 3.1 Cypher 查询

```cypher
// 在"太湖"中查找节点，包含引用的节点
MATCH (lake:Lake {name: "太湖"})<-[:BELONGS_TO]-(localNode:Node)
OPTIONAL MATCH (localNode)-[:REFERENCES]->(externalNode:Node)
WHERE externalNode.state <> 'ERASED'
RETURN localNode, externalNode;
```

### 3.2 前端表现

- 引用节点显示**半透明** + ⛵ 小船图标
- Hover 提示："此节点源自'西湖'"

### 3.3 权限控制

- 引用节点**只读**
- 用户尝试修改：提示"**此节点源自外湖，是否创建副本？(Clone)**"
- 详细权限：[G1 §5.2](G1-数据模型与权限设计.md)

### 3.4 GHOST 转换（源节点被删时）

详见 [G1 §三状态机](G1-数据模型与权限设计.md)：源节点 ERASED 时，引用方看到"游魂节点"（半透明灰、内容快照保留 30 天），并收到通知。

## 四、合流模式（MERGE）实现

### 4.1 逻辑流程

1. 创建新湖（如"江湖"）或指定目标湖
2. 将原湖 A、B 中所有 `:Node` 的 `[:BELONGS_TO]` 重定向到新湖
3. 保留原有 `[:RELATES_TO]` 关系
4. 在原湖创建 `[:MERGED_INTO]` 历史记录

### 4.2 伪代码

```python
def merge_lakes(lake_ids, target_lake_id):
    for lake_id in lake_ids:
        nodes = neo4j.run(
            "MATCH (n)-[:BELONGS_TO]->(:Lake {id: $lid}) RETURN n",
            lid=lake_id
        )
        for node in nodes:
            node.run("MATCH (n)-[r:BELONGS_TO]->() DELETE r")
            node.run(
                "MATCH (n), (t:Lake {id: $tid}) CREATE (n)-[:BELONGS_TO]->(t)",
                tid=target_lake_id
            )
    create_merge_history(lake_ids, target_lake_id)
```

> 重量级操作，必须异步 + 可回滚。

## 五、UI/UX 交互

### 5.1 拖拽连通

- 操作：将一个湖泊（如"西湖"）拖到另一个湖泊（如"太湖"）的画布上
- 反馈：**"已开通西湖 ⇋ 太湖 的运河。"**
- 来自西湖的节点以特殊样式（带波纹边框）显示

### 5.2 地下暗河探测（AI 辅助）

- 用户点击**探源**：除本湖外，自动扫描**其他湖泊**中相似度高的节点
- 提示：**"在'玄武湖'中发现强关联节点，是否开通暗河引入？"**

## 六、风险与规避

### 6.1 循环引用

- 防止"西湖"引用"太湖"，同时"太湖"又引用"西湖"
- 解决：创建 `[:REFERENCES]` 时检查反向路径，存在则提示"合流可能更合适"

### 6.2 性能

- 跨湖查询增加一次 Hop
- 解决：Neo4j APOC 库高效路径遍历，应用层 Redis 缓存
