> - 必须按照文档执行 #file:copilot.instructions.md

**解读**：
- 转向不是否定 Python 实现，而是验证完成后选择更适合长期演进的栈
- "执行 1 2 3" 在新上下文下重新映射为：补约束文档 / 启动 Go 方案 / 更新 AGENTS.md

**决策**（决策者·项目经理）：
1. 本轮**只做文档**，不写一行 Go 代码（中间件信息未到，写了必返工）

**行动**：

## 2025-Q4 · AskQuestions 缺失固定选项违规

**触发**：上轮 AskQuestions 未包含 `必须按照文档执行 #file:copilot.instructions.md ` 固定选项，被老板在 freeText 中点名。

**反馈**：老板在自由输入中显式要求"必须按照文档执行"。

**解读**：违反了 `copilot.instructions.md` §AskQuestions Requirements 的硬性要求——每次询问都必须包含该固定选项。

**决策**：
- 立即复述规则原文（已在本轮回复中执行）
- 本轮起每次 AskQuestions 必带固定选项 + ≥3 预设 + 允许多选 + 允许自由输入
- 自查清单写入下方"会话收口前自查"

**行动**：
- ✅ 本日志记录违规事实
- ✅ 会话记录写入 `/memories/session/ripple-go-pivot.md`

**预防**：每次准备 AskQuestions 前，三项自查：
1. 是否包含固定选项？
2. 是否 ≥ 3 预设？
3. 是否明确指向下一步任务？
任一项缺失 → 立即补齐，禁止收口。

## 2026-04-22 · Phase F+G 全跑通 + 多 LLM provider 抽象触发

**触发**：本轮完成 Phase F（Outbox saga）+ Phase G（造云/AI Weaver/ZhipuAI）后，老板在 AskQuestions freeText 反馈：
> zhipu ApiKey: ********（已脱敏）
> 这里我认为架构需要升级：未来肯定不是只支持一家 llm

**反馈**：
1. 旧 key 401，提供新 key（已脱敏）→ 重跑 e2e 成功，ZhipuAI 实际产出 5 个 MIST 节点
2. **结构性架构反馈**：当前 `internal/llm/zhipu.go` 直接耦合，未来要支持多家 LLM（ZhipuAI / OpenAI / Anthropic / 本地模型...）

**解读**：
- 配置层问题（key）即时修复无需讨论
- 架构层问题（多 provider）属"核心架构变更"——按 AGENTS.md "何时不该主动行动" 原则，不直接写代码，必须经过十角色（接口设计师 + 正方 + 反方）评审
- 当前 `llm.Client` 接口 `Generate(ctx, prompt, n) ([]string, error)` 已经是抽象的雏形，但有四个不足：
  1. 仅支持文本生成，未来要支持图像/音频（多模态）
  2. 没有路由策略（按租户/按任务类型/按成本选 provider）
  3. 没有 fallback / 限流 / 计费追踪
  4. 单 endpoint 硬编码，无法热切换

**决策**（决策者·待评审）：
- 本轮 Phase G 已 push（`63fcdaf`）保留单 provider 实现，作为 baseline
- 下一轮启动十角色对"LLM Provider 抽象层"做正式提案，不写代码
- 候选方向：`llm.Provider` 接口 + `llm.Router` + 配置驱动注册 + 多模态 `Modality` 枚举

**行动**：
- ✅ 更新 .env: `RIPPLE_ZHIPU_API_KEY=********`（实际值仅存于本地未追踪 .env）
- ✅ 重跑 cloud e2e 成功（task `41eb8bd8` → 5 MIST nodes）
- ⏸ 下一轮：接口设计师产出 LLM Provider 抽象方案，正方/反方对线
- ⏸ 评审通过后再写代码

**预防**：
- 任何新增对外 LLM/AI 第三方依赖时，先检查 AGENTS.md "何时不该主动行动" → 涉及核心架构必须停手提方案
- 在 `系统约束规约.md` 增补条款：所有 LLM 调用必须经统一 Router 入口，禁止业务层直接 import 厂商 SDK
