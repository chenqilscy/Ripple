# 系统约束规约 · Ripple 青萍

> **本文档定义"什么不可以"和"边界在哪里"。**
> 任何实现（无论 Python/Go/Rust）必须满足以下约束。约束变更必须经决策者裁决。

## 1. 性能与容量约束（HARD）

### 1.1 响应时间 SLO

| 路径类型 | P50 | P95 | P99 | 备注 |
|---------|-----|-----|-----|------|
| 健康检查 `/health` | < 5ms | < 10ms | < 20ms | 不读 DB |
| 鉴权 `/auth/*` | < 50ms | < 150ms | < 300ms | 含 bcrypt |
| 节点读 `GET /nodes/*` | < 30ms | < 80ms | < 200ms | Neo4j 命中索引 |
| 节点写 `POST /nodes` | < 80ms | < 200ms | < 500ms | 含 outbox 写入 |
| WebSocket 广播延迟 | < 50ms | < 150ms | < 500ms | 同湖内 |
| AI Weaver 织网 | < 500ms | < 2s | < 5s | 后台任务，不阻塞 API |

### 1.2 吞吐与并发

| 指标 | M1 目标 | M2 目标 | 上限假设 |
|------|--------|--------|---------|
| 并发用户 | 100 | 1000 | 10000 |
| 单湖泊节点数 | 1000 | 10000 | 50000（超此分湖） |
| 同湖在线协作者 | 10 | 50 | 100 |
| 请求 QPS | 50 | 500 | 5000 |
| WebSocket 连接 | 200 | 2000 | 20000 |

### 1.3 资源上限

- 单节点 content：≤ 10 KB
- 单次 API 请求体：≤ 1 MB
- 文件上传（M2）：≤ 10 MB（图）/ 100 MB（音频）
- WebSocket 消息：≤ 64 KB

## 2. 数据约束（HARD）

### 2.1 持久化分工（不可混淆）

| 数据类别 | 存储 | 原因 |
|---------|------|------|
| 用户账号 / 鉴权 / 审计 / outbox | PostgreSQL | ACID + 关系完整性 |
| 节点 / 湖泊 / 边 / 关系图谱 | Neo4j | 图遍历是核心查询 |
| 会话 / 限流 / Pub/Sub | Redis | 低延迟 + TTL 原生 |
| 大文件（图/音/视） | S3 / MinIO | 不入 DB |
| Embedding 向量（M2） | 待定（pgvector 或 Qdrant） | M1 用内存 |

### 2.2 数据生命周期（状态机）

```
MIST (TTL 7d) → DROP → FROZEN → VAPOR (TTL 30d) → ERASED
                                       ↓
                                   GHOST（跨湖引用保留）
```

- **MIST 7 天**：到期自动 ERASED（除非凝露 → DROP）
- **VAPOR 30 天**：到期物理删除（除非用户在迷雾区还原）
- **ERASED 后保留 90 天审计日志**，之后 cron 物理清除审计
- **不可删除**：lake_memberships（OWNER 转让而非删除）

### 2.3 数据所有权与还原

- 用户对自己创建的节点拥有**永久还原权**（直到 ERASED 物理删除）
- 用户**任何时候**可导出自己的全部数据为 JSON（GDPR 数据可携权）
- 删除账号 = 30 天软删，期间可恢复，之后级联 ERASED 全部数据

### 2.4 一致性

- 同一事务内：PG 强一致
- 跨 PG / Neo4j：最终一致（通过 outbox + Worker 异步同步）
- 跨节点的 WebSocket 广播：at-least-once，前端去重

## 3. 安全约束（HARD）

### 3.1 鉴权 / 鉴别

- 密码：bcrypt cost ≥ 12，硬截断 72 字节
- JWT：HS256（M1）→ RS256（M2 微服务化）
- access_token TTL：1 小时；refresh_token TTL：30 天
- WebSocket 鉴权：query 参数 `?token=`，握手时校验
- 限流：登录 5 次/分钟/IP；注册 3 次/小时/IP；API 60 次/分钟/用户

### 3.2 鉴权矩阵

```
OWNER (3) > NAVIGATOR (2) > PASSENGER (1) > OBSERVER (0)

操作                  最低 role
--------------------- ---------
读节点                OBSERVER
创建/编辑节点         PASSENGER
蒸发自己创建的节点    PASSENGER
蒸发他人节点          OWNER
还原他人节点          OWNER
邀请成员              NAVIGATOR
转让 OWNER            OWNER（仅一人）
删除湖                OWNER
```

### 3.3 数据隔离

- **跨用户隔离**：任何 service 函数必须接收 `actor_user_id` 参数并校验
- **跨湖隔离**：Node 必须 BELONGS_TO Lake，跨湖查询必须显式声明
- **管理员视图**：禁止建造"上帝视角"绕过权限的接口

### 3.4 输入校验

- 所有外部输入必须 schema 校验（Pydantic / go-playground/validator）
- HTML/Markdown 渲染必须经过 DOMPurify（前端）+ bluemonday（Go）/ bleach（Python）
- SQL：禁止字符串拼接，必须参数化
- Cypher：使用 `$param` 占位符
- 文件上传：白名单 MIME + 重命名 UUID + 病毒扫描（M2）

### 3.5 隐私与合规

- 默认数据本地化（中国境内：阿里云杭州；海外：AWS us-west-2）
- 不向第三方 LLM 上传用户原文（除非用户显式授权）
- AI Weaver 的 Embedding 优先本地模型；远程模型必须脱敏

## 4. 可用性约束（HARD/SOFT）

| 指标 | M1 (HARD) | M2 (HARD) | 长期 (SOFT) |
|------|-----------|-----------|-------------|
| 月度可用性 | 99.0% (≤ 7.2h 停机) | 99.5% | 99.9% |
| RPO（数据丢失窗口） | 24h（每日全备） | 1h | 5min |
| RTO（恢复时间） | 4h | 1h | 15min |
| 计划维护窗口 | 周日 02:00-06:00 | 滚动升级 | 零停机 |

### 4.1 故障域

- 单点故障可接受范围：M1 单节点；M2 起必须 ≥ 2 副本
- DB 备份：每日全备 + WAL 持续归档至 S3
- 跨区灾备：M3 启动

### 4.2 降级策略

- AI Weaver 故障 → 节点照常创建，仅不自动织网（用户手动织）
- Neo4j 故障 → 只读模式（PG 仍可登录，前端显示"湖面凝结中"）
- Redis 故障 → 限流降级到本地内存（容许 5 倍流量穿透）

## 5. 可观测性约束（HARD）

- 所有 API 请求必须有 `X-Request-ID`（入站生成或透传）
- 日志：结构化 JSON（生产）；human-readable（开发）
- 关键事件必须打 metric：`node.created`、`node.evaporated`、`weaver.matched`、`auth.failed`
- 错误必须分级：DEBUG / INFO / WARN / ERROR / FATAL
- 不允许 silent error（违反底线 #2）

## 7. 技术栈约束（HARD）

### 7.1 强制选型

| 层 | 选型 | 禁用 |
|----|------|------|
| 后端语言 | **Go 1.23+**（M1 之后） | Node.js / Java / Rust |
| 前端 | TypeScript + React 18 + Vite | Vue / Angular / 纯 JS |
| 渲染 | Three.js + WebGL2 | Canvas2D（除占位）/ Babylon |
| 关系库 | PostgreSQL 16+ | MySQL / SQLite（生产） |
| 图库 | Neo4j 5+ | Neptune / TigerGraph |
| 缓存 | Redis 7+ | Memcached |
| 对象存储 | S3 兼容（MinIO 自部署） | 本地磁盘 |
| 容器 | Docker + Compose（M1）→ K8s（M2） | 裸机 |

### 7.2 Python 资产说明

- M1 Python 实现 (`backend/`) 保留至 `v0.1-python-archive` tag，作为参考实现
- 不再向 Python 后端提交新功能
- Go 实现完成同等功能后，Python 目录归档至 `archive/backend-python/`

### 7.3 库选型偏好（Go）

- HTTP 路由：Chi 或 Echo
- DB：pgx/v5 + sqlc（不用 ORM）
- Neo4j：neo4j-go-driver/v5
- Redis：go-redis/v9
- WebSocket：nhooyr.io/websocket
- JWT：golang-jwt/jwt/v5
- 配置：envconfig
- 日志：zerolog
- 测试：标准库 testing + testify
- Mock：counterfeiter 或手写

## 8. 用户体验约束（HARD）

### 8.1 不可破坏的承诺

1. **绝不丢数据**：用户的节点必须能完整导出（JSON / Markdown）
2. **绝不静默错误**：失败必须有可读提示 + 日志可追
3. **绝不破坏兼容**：v1 API 一旦发布，至少维护 12 个月
4. **绝不广告骚扰**：免费版可有功能限制，但禁止弹窗/插页广告
5. **永久免费版**：必须保留可用核心功能（创建湖、500 节点、单人）

### 8.2 设计哲学约束

#### 8.2.1 视觉与动效

- **目标帧率**：60 FPS @ 1080p；不允许 jank（> 50ms 主线程阻塞）
- **缓动函数**：禁用线性（linear），统一使用 `cubic-bezier(0.22, 0.61, 0.36, 1)`（"水波"曲线）或 spring 物理
- **持续时间**：微交互 150-300ms；场景切换 400-600ms；超过 800ms 必须有进度反馈
- **色板锁定**（参见 `docs/pinpai/`）：
  - 主色：青萍绿 `#2E8B57`、湖水蓝 `#001E2B`、暮色珊瑚 `#FF7F50`
  - 中性：墨白 `#E6F2EE`、薄雾灰 `#A8B8B0`
  - 禁用：纯黑 `#000`、纯白 `#FFF`、霓虹色（饱和度 > 80%）
- **粒子密度**：单视口 ≤ 2000 粒子；超出降级为 sprite

#### 8.2.2 文案与命名

- **核心隐喻保持一致**（不可乱串）：
  - 想法 = 涟漪 / 浮萍 / 水滴
  - 容器 = 湖泊 / 池塘
  - 临时草稿 = 迷雾（mist）
  - 删除 = 蒸发（evaporate）→ 还原 = 凝露（condense / restore）
  - 关联 = 织网（weave）/ 牵丝
  - 共享 = 引流 / 通渠
- **禁用互联网黑话**：`dashboard / feed / timeline / engagement` → `案台 / 水文 / 流年 / 共鸣`
- **错误文案**：必须给出**下一步操作**，不只是报错
  - ❌ "Network error"
  - ✅ "湖面起雾了，请稍后再试。如持续 5 分钟仍无法连接，[查看状态页]"
- **空状态文案**：必须留白 + 诗意引导
  - 示例："此处风平浪静，等待第一缕青萍。"

#### 8.2.3 加载与离线

- **首屏目标**：FCP < 1s（光纤）；TTI < 2s
- **骨架屏**：禁用通用灰块；统一用"水波纹"loading
- **离线**（M2 起）：
  - 核心读路径必须支持离线缓存（Service Worker + IndexedDB）
  - 离线时新建节点进入"云霓本地池"，恢复网络后批量同步
  - 离线 UI 必须有明确指示（顶栏"当前离线 · X 个待同步"）

#### 8.2.4 输入与可达性

- **键盘可用**：所有可点击元素必须可 Tab + Enter 触发
- **快捷键**（M2）：
  - `Cmd/Ctrl + N`：新节点
  - `Cmd/Ctrl + K`：搜索 / 命令面板
  - `Esc`：返回 / 退出当前模式
  - `空格 + 拖动`：平移湖面
- **a11y**：
  - 对比度 ≥ 4.5:1（正文）/ 3:1（大字）
  - 所有图像必须有 alt
  - 动效必须可关闭（`prefers-reduced-motion`）
- **多端响应式**：≥ 360px 宽度可用；触屏手势 = 双指缩放湖 / 长按节点弹菜单

#### 8.2.5 反"成瘾"设计承诺

青萍**主动拒绝**以下"互联网毒性"模式：

- ❌ 无限下拉信息流
- ❌ 红点 / 数字徽章诱导点击
- ❌ "X 人正在查看" / "限时优惠" 等 FOMO 触发
- ❌ 通过推送通知召回（除非用户主动订阅特定湖泊更新）
- ❌ 排行榜 / 等级 / 段位 / 连续天数

**允许的反馈机制**：
- ✅ 节点的"涟漪扩散"动画（视觉愉悦但无诱导）
- ✅ 周报 / 月报（用户可关闭）：本月你的湖泊涌出了 N 朵浮萍
- ✅ 织网建议（被动展示，不弹窗）

#### 8.2.6 沉浸与专注

- **默认无干扰**：不弹通知 / 不显示在线人数 / 不显示其他用户头像（除非进入协作模式）
- **专注模式**：单击湖面空白处 → UI 全部淡出，仅留湖面与浮萍
- **晨昏制**：根据用户本地时间调整湖面色温（晨青 / 午翠 / 暮珊瑚 / 夜墨）

### 8.3 命名规范（代码 + 资源）

| 对象 | 规范 | 示例 |
|------|------|------|
| 文件名（前端组件） | PascalCase.tsx | `LakeCanvas.tsx` |
| 文件名（Go 包） | snake_case.go | `node_service.go` |
| Go 包名 | lowercase | `package node` |
| Go 接口 | -er 后缀 | `NodeReader`, `LakeWriter` |
| Go 结构体 | PascalCase | `LakeMembership` |

| API 字段（JSON） | snake_case | `created_at`, `lake_id` |
| 数据库表 | snake_case 复数 | `lake_memberships` |
| Neo4j 节点标签 | PascalCase | `:Lake`, `:Node` |
| Neo4j 关系类型 | UPPER_SNAKE | `:BELONGS_TO`, `:RELATES_TO` |
| 环境变量 | UPPER_SNAKE | `RIPPLE_PG_DSN` |
| 事件类型（outbox） | dot.notation | `node.condensed`, `lake.member.invited` |

## 9. 审查约束（HARD）

每次 PR 合并前，反方必须按 `AGENTS.md` 五轮审查执行：
1. 逻辑正确性
2. 边界与异常
3. 架构一致性
4. 安全与隔离
5. 性能与资源

任一轮不过 → 不准合并。
