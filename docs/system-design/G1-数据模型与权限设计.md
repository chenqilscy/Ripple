# G1 · 数据模型与权限设计

**版本：** v1.0
**日期：** 2026‑04‑21
**适用对象：** 后端 / DBA / 安全
**地位：** **权威 Schema 文档**——所有其他设计文档对 Node/Edge/Lake/User 的描述以本文档为准。冲突即返工。

---

## 一、设计原则

1. **双库架构**：
   - **Neo4j**：图数据（Lake / Node / Edge / 引用 / 合流历史）
   - **PostgreSQL**：关系数据（User / Membership / Token / 审计日志 / 文件元数据 / 漂流瓶）
2. **软删一切**：所有用户数据均使用 `deleted_at` 软删，30 天 TTL 后由后台 Job 物理删除。
3. **不可变审计**：写操作产生不可变审计事件，进入只追加日志。
4. **权限三层**：湖泊级（Lake-level） → 节点级（Node-level，可选）→ 引用级（Reference-level，跨湖只读契约）。

## 二、Neo4j 图 Schema（权威）

### 2.1 节点标签 (Labels)

```cypher
// 用户引用（Neo4j 内仅存 ID，详细信息在 PostgreSQL）
(:UserRef {id: UUID, display_name: STRING})

// 湖泊
(:Lake {
  id: UUID,
  name: STRING,            // 西湖、玄武湖、用户自定义
  theme: STRING,           // 西湖→文艺；玄武湖→历史；可空
  visibility: STRING,      // PRIVATE | TEAM | PUBLIC
  created_at: DATETIME,
  deleted_at: DATETIME?    // 软删
})

// 浮萍节点（核心实体）
(:Node {
  id: UUID,
  lake_id: UUID,           // 冗余字段，便于查询过滤
  type: STRING,            // TEXT | IMAGE | VOICE | DOC_CHUNK | MIST | BOTTLE | GHOST
  content: STRING,         // 主内容（文本或元数据 JSON）
  embedding: LIST<FLOAT>,  // 1536 维向量（OpenAI text-embedding-3-small）
  state: STRING,           // MIST | DROP | FROZEN | VAPOR | ERASED
  position_x: FLOAT,
  position_y: FLOAT,
  created_by: UUID,        // UserRef.id
  created_at: DATETIME,
  updated_at: DATETIME,
  deleted_at: DATETIME?,   // VAPOR 状态时设值，30 天后物理删
  ttl_at: DATETIME?,       // 仅 MIST 节点有，7 天默认
  asset_ref: UUID?         // 关联的文件资产 ID（如图片/语音/PDF）
})

// 资产快照（用于冰山）
(:Snapshot {
  id: UUID,
  iceberg_id: UUID,
  taken_at: DATETIME,
  node_count: INT,
  edge_count: INT
})

// 冰山
(:Iceberg {
  id: UUID,
  source_lake_id: UUID,
  title: STRING,
  description: STRING,
  visibility: STRING,      // PRIVATE | TEAM | PUBLIC
  created_at: DATETIME,
  deleted_at: DATETIME?
})
```

### 2.2 关系类型 (Relationships)

| 关系 | 起→止 | 属性 | 含义 |
| :--- | :--- | :--- | :--- |
| `[:BELONGS_TO]` | `Node → Lake` | — | **唯一性约束**：一个节点只属于一个湖 |
| `[:RELATES_TO]` | `Node → Node` | `{type, weight, reasoning, source, created_at}` | "暗流"。`type` ∈ {METAPHOR, SIMILAR_TO, CAUSES, CONTRASTS, PART_OF}；`source` ∈ {USER, AI} |
| `[:REFERENCES]` | `Node → Node` | `{from_lake, permission, created_at}` | 跨湖引用（运河模式），目标只读 |
| `[:MERGED_FROM]` | `Lake → Lake` | `{merged_at, by_user}` | 合流历史 |
| `[:CREATED_BY]` | `Node → UserRef` | — | 创作归属（用于权限） |
| `[:SNAPSHOT_OF]` | `Snapshot → Lake` | — | 快照与湖泊的关联 |
| `[:CONTAINS]` | `Snapshot → Node` | `{node_version}` | 快照冻结的节点集 |
| `[:DERIVED_FROM]` | `Iceberg → Snapshot` | — | 冰山源自哪个快照 |
| `[:GHOST_OF]` | `Node → Node` | `{erased_at, original_lake_id, content_snapshot}` | 源节点被彻底删后的"游魂" |

### 2.3 索引与约束

```cypher
// 唯一性
CREATE CONSTRAINT lake_id_unique     IF NOT EXISTS FOR (l:Lake)     REQUIRE l.id IS UNIQUE;
CREATE CONSTRAINT node_id_unique     IF NOT EXISTS FOR (n:Node)     REQUIRE n.id IS UNIQUE;
CREATE CONSTRAINT iceberg_id_unique  IF NOT EXISTS FOR (i:Iceberg)  REQUIRE i.id IS UNIQUE;

// 节点-湖泊唯一关系
CREATE CONSTRAINT node_belongs_to_one_lake IF NOT EXISTS
  FOR ()-[r:BELONGS_TO]-() REQUIRE r IS UNIQUE;

// 索引
CREATE INDEX node_lake_state IF NOT EXISTS FOR (n:Node) ON (n.lake_id, n.state);
CREATE INDEX node_created_at IF NOT EXISTS FOR (n:Node) ON (n.created_at);
CREATE INDEX node_ttl IF NOT EXISTS FOR (n:Node) ON (n.ttl_at);

// 向量索引（探源、AI 推荐用）
CREATE VECTOR INDEX node_embedding IF NOT EXISTS
  FOR (n:Node) ON (n.embedding)
  OPTIONS { indexConfig: { `vector.dimensions`: 1536, `vector.similarity_function`: 'cosine' } };
```

## 三、节点生命周期状态机（统一收敛点）

> **修复 R3.1 / R3.2**：统一以前的"软删/迷雾区/Disabled/蒸发"四套术语。

```
       ┌─────────────────────────────────────────────────┐
       │                                                  │
[Cloud 录入] ──► MIST ──[凝露]──► DROP ──[固形]──► FROZEN │
                  │                │                  │   │
                  │                │   [蒸发]          │   │
            [TTL 7天]              ▼                  ▼   │
                  │              VAPOR ◄────────[蒸发]────┤
                  ▼               │                       │
              ERASED ◄─[TTL 30天]─┘                       │
              (物理删)                                    │
                                                         │
              GHOST ◄─[源节点被引用方继续 ERASED]──────────┘
              (游魂，仅快照)
```

| 状态 | 含义 | 可见性 | 可编辑 | 触发条件 |
| :--- | :--- | :--- | :--- | :--- |
| `MIST` | 雾气节点（云霓待凝露区） | 仅创建者 | 否 | 云霓离线/在线录入 |
| `DROP` | 浮萍节点（造浪池常态） | 按权限 | 是 | MIST 经"凝露"动作 |
| `FROZEN` | 已固形 | 按权限 | 否（需先解冻） | 手动"固形"或纳入冰山快照 |
| `VAPOR` | 已蒸发，迷雾区可见 | 仅原创者 + 湖泊 Owner | 否 | "蒸发"动作 |
| `ERASED` | 物理删除 | 不可见 | — | VAPOR 满 30 天 / 强制清除 |
| `GHOST` | 跨湖被引用过的已 ERASED 节点 | 引用方半透明显示 | 否 | 见 R4.2 |

## 四、PostgreSQL 关系 Schema

```sql
-- 用户表
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email         CITEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,            -- argon2id
  display_name  VARCHAR(64) NOT NULL,
  avatar_url    TEXT,
  locale        VARCHAR(8)  DEFAULT 'zh-CN',
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at    TIMESTAMPTZ
);

-- 湖泊成员关系
CREATE TABLE lake_memberships (
  lake_id     UUID NOT NULL,
  user_id     UUID NOT NULL REFERENCES users(id),
  role        VARCHAR(16) NOT NULL,       -- OWNER(舟子) | NAVIGATOR(副舟子) | PASSENGER(渡客) | OBSERVER(观潮)
  invited_by  UUID REFERENCES users(id),
  joined_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at  TIMESTAMPTZ,
  PRIMARY KEY (lake_id, user_id)
);
CREATE INDEX idx_membership_user ON lake_memberships(user_id) WHERE deleted_at IS NULL;

-- 长期会话 / Refresh Token
CREATE TABLE auth_tokens (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id),
  token_hash  TEXT NOT NULL,             -- 仅存 SHA-256，明文不入库
  device      VARCHAR(64),
  ip_inet     INET,
  expires_at  TIMESTAMPTZ NOT NULL,
  revoked_at  TIMESTAMPTZ,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 文件资产元数据（详见 G4）
CREATE TABLE assets (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_id      UUID NOT NULL REFERENCES users(id),
  bucket        VARCHAR(64) NOT NULL,
  object_key    TEXT NOT NULL,
  mime_type     VARCHAR(128) NOT NULL,
  size_bytes    BIGINT NOT NULL,
  sha256        CHAR(64) NOT NULL,
  status        VARCHAR(16) NOT NULL,    -- UPLOADING | READY | PROCESSING | FAILED | DELETED
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at    TIMESTAMPTZ,
  UNIQUE (bucket, object_key)
);

-- 漂流瓶（公开分享回流）
CREATE TABLE bottles (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  share_link_id UUID NOT NULL,
  iceberg_id    UUID NOT NULL,
  sender_fingerprint CHAR(64) NOT NULL, -- 防滥用（IP+UA+随机种子 hash）
  message       VARCHAR(500) NOT NULL,
  status        VARCHAR(16) NOT NULL,   -- PENDING_REVIEW | APPROVED | REJECTED | RECEIVED
  reviewed_at   TIMESTAMPTZ,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_bottles_anti_spam ON bottles(sender_fingerprint, iceberg_id, created_at);

-- 公开分享链接
CREATE TABLE share_links (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  iceberg_id    UUID NOT NULL,
  created_by    UUID NOT NULL REFERENCES users(id),
  visibility    VARCHAR(16) NOT NULL,   -- BLURRED | CLEAR
  password_hash TEXT,                   -- 可空
  expires_at    TIMESTAMPTZ NOT NULL,
  revoked_at    TIMESTAMPTZ,
  view_count    BIGINT DEFAULT 0,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 审计日志（不可变，按月分区）
CREATE TABLE audit_events (
  id          BIGSERIAL,
  occurred_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  actor_id    UUID,
  action      VARCHAR(64) NOT NULL,    -- LAKE_CREATE | NODE_EVAPORATE | LINK_CREATE | ...
  target_kind VARCHAR(32) NOT NULL,    -- LAKE | NODE | EDGE | ICEBERG | SHARE_LINK
  target_id   UUID,
  payload     JSONB,
  ip_inet     INET,
  user_agent  TEXT,
  PRIMARY KEY (id, occurred_at)
) PARTITION BY RANGE (occurred_at);
```

## 五、权限矩阵

### 5.1 湖泊角色

| 角色 | 创建节点 | 蒸发自己节点 | 蒸发他人节点 | 还原迷雾区 | 邀请成员 | 决堤口 | 跨湖引用 |
| :--- | :-: | :-: | :-: | :-: | :-: | :-: | :-: |
| **OWNER** (舟子) | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **NAVIGATOR** (副舟子) | ✅ | ✅ | ✅ | ✅ | ✅ | — | ✅ |
| **PASSENGER** (渡客) | ✅ | ✅ | — | 仅自己创建的 | — | — | 仅自己创建的 |
| **OBSERVER** (观潮) | — | — | — | — | — | — | — |

> **修复 R1.2**：还原迷雾区的权限分两级——普通渡客只能还原自己创建的节点，OWNER/NAVIGATOR 可还原任何节点。

### 5.2 跨湖引用 (REFERENCES) 权限

- 创建引用：源湖必须对当前用户**至少 OBSERVER 可见**
- 引用节点对目标湖成员**始终只读**
- 源湖所有者删除节点时：触发 `GHOST_OF` 转换（详见 §三状态机）

### 5.3 JWT Claims 标准

```json
{
  "sub": "<user_id>",
  "email": "...",
  "roles": ["user"],
  "lakes": [
    {"id": "<lake_id>", "role": "OWNER"},
    {"id": "<lake_id>", "role": "PASSENGER"}
  ],
  "iat": 1714000000,
  "exp": 1714003600,
  "jti": "<token_id>"
}
```

- Access Token：1 小时；Refresh Token：30 天
- `lakes` claim 不超过 50 项，超出则懒加载
- 服务端验证后**必须**与 PostgreSQL `lake_memberships` 二次校对（防 token 过期变更）

## 六、关键查询模式

### 6.1 获取湖内可见节点（含跨湖引用）
```cypher
MATCH (lake:Lake {id: $lakeId})<-[:BELONGS_TO]-(local:Node)
WHERE local.state IN ['DROP', 'FROZEN', 'MIST']
OPTIONAL MATCH (local)-[r:REFERENCES]->(ext:Node)
WHERE ext.state <> 'ERASED'
RETURN local, collect({ref: ext, perm: r.permission}) AS refs;
```

### 6.2 还原迷雾区节点（带连边）
```cypher
MATCH (n:Node {id: $nodeId})
WHERE n.state = 'VAPOR' AND n.deleted_at IS NOT NULL
SET n.state = 'DROP', n.deleted_at = NULL
WITH n
// 入向暗流自动复活
MATCH (n)-[r:RELATES_TO]-(other:Node)
WHERE other.state <> 'ERASED'
RETURN n, count(r) AS restored_edges;
```

### 6.3 探源（向量近似 + 跨湖）
```cypher
CALL db.index.vector.queryNodes('node_embedding', 20, $queryEmbedding)
YIELD node, score
WHERE node.state IN ['DROP', 'FROZEN']
  AND (
    EXISTS { MATCH (node)-[:BELONGS_TO]->(:Lake {id: $currentLakeId}) }
    OR EXISTS {
      MATCH (node)-[:BELONGS_TO]->(other:Lake)
      WHERE other.visibility IN ['PUBLIC']
        OR EXISTS { ... 当前用户成员关系 ... }
    }
  )
RETURN node, score ORDER BY score DESC LIMIT 10;
```

## 七、迁移与版本

- 所有 Schema 变更走 **Alembic（PostgreSQL）+ Neo4j Migration Tool**（如 [neo4j-migrations](https://michael-simons.github.io/neo4j-migrations/)）
- 迁移脚本命名：`YYYYMMDDHHMM_<scope>_<desc>.{sql,cypher}`
- 灰度策略：所有破坏性变更（删字段/改类型）必须经历"双写期 → 切换期 → 清理期"三阶段，每阶段最少 7 天
