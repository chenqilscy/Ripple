## 1. 已交付能力

### F1 造浪池雏形（先于本节，已在 M1/M2 早期完成）
- 节点 CRUD + DROP/MIST/CLOUD/VAPOR 状态机
- 边 CRUD + 自环防护 + 重复防护
- 蒸发/凝露/恢复
- 造云 worker pool + LLM Router（zhipu/openai/deepseek/volc/minimax/openai-compat 6 家 provider）

### F2 邀请协作（M2 中期）
- `invites` 表（PG）+ `InviteRepository` + `InviteService`
- 端点：`POST/GET /lakes/{id}/invites`、`POST /invites/preview`、`POST /invites/{token}/accept`、`DELETE /invites/{id}`
- 前端：sidebar OWNER 可见「邀请」按钮、prompt 流创建/撤销、URL `?invite=token` 自动 preview+accept
- 集成测试：`TestIntegrationInviteFlow`

### F3 节点编辑历史（M2 后期）
- `node_revisions` 表（PG，事件溯源）+ `NodeRevisionRepository`
- `NodeService.UpdateContent` 幂等（同内容不增 rev）+ `Rollback(targetRev)`
- 端点：`PUT /nodes/{id}/content`、`GET /nodes/{id}/revisions[?limit]`、`GET /nodes/{id}/revisions/{rev}`、`POST /nodes/{id}/rollback`
- 前端：节点卡片 ✎ 编辑 / ⟲ 历史按钮，prompt 显示 rev 列表 + 输入 rev 号回滚
- 单元测试：6 个（含权限/边界/幂等/回滚）
- 集成测试：`TestIntegrationNodeRevisionFlow`

### F4 在线状态（M2 收官）
- 新建 `internal/presence` 包，Service{rdb, mem, broker, ttl}
- Redis ZSET 存 `lake:{id}:presence`，score=过期时间戳；按 score 过滤过期 + ZRemRangeByScore 清理
- WS Handler 自动 Join/Heartbeat/Leave，前端 30s ping 续期
- 端点：`GET /lakes/{id}/presence`
- 前端：顶部在线圆标列表（前 8 + 溢出指示）
- 单元测试：4 个（含 TTL/边界/内存模式）
- 集成测试：`TestIntegrationPresenceFlow`（需 Redis）

### LLM Provider 全栈（M2 全程并行交付）
- 已实现：zhipu / openai / deepseek / volc / minimax / openai-compat（共 6 家）
- 协议归一：5 家走 `OpenAICompatClient`，zhipu 单独
- 文档：[LLM-Provider-接入手册.md](LLM-Provider-接入手册.md)

## 3. 关键决策与变更

| 决策 | 解释 | 沉淀文档 |
|------|------|---------|
| 节点编辑用事件溯源（不覆盖） | 满足"绝不丢数据"底线；回滚等价于"再追加一条等同旧版本的 rev" | `内部决策` |
| Presence 走 Redis ZSET 而非 Pub/Sub | TTL 自然过期 + 列表查询 O(log N) + 不需要订阅历史 | 本文档 |
| Claude Code 暂停在 build tag | ToS 合规未核 + 单订阅商业复用风险 | [LLM-Provider-接入手册.md](LLM-Provider-接入手册.md) §4 |
| 前端 F2/F3 用 prompt 占位 | M2 优先打通后端，UI 在 M3 升级（见 [技术债清单.md](技术债清单.md) TD-010） | 本文档 |
| Race 修复：ClaimNext 返回拷贝 | 否则 worker 持有 map 内指针与 mu 锁外读测试并发 | [自学习日志.md](自学习日志.md) |
