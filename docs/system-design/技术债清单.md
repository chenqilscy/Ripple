## P0（阻断后续路线）

### TD-001 OpenAI 兼容客户端不支持流式输出 ✅ 已偿还（2025-Q4）
- **影响**：长输出（>1K tokens）整体等待，前端 UX 差
- **位置**：[openai_compat.go](backend-go/internal/llm/openai_compat.go) `Generate()`
- **解决**：新增可选接口 `StreamProvider`（[stream.go](backend-go/internal/llm/stream.go)）；`OpenAICompatClient.GenerateStream` 实现 SSE 解析（[openai_compat_stream.go](backend-go/internal/llm/openai_compat_stream.go)），单元测试覆盖正常/4xx/非文本三类场景。Router 集成留待 M3 PERMA。

### TD-002 无 Provider 级速率限制 ✅ 已偿还（2025-Q4）
- **影响**：用户高频造云会触发上游 429，缺少本地节流
- **位置**：[router.go](backend-go/internal/llm/router.go)
- **解决**：新增 `RateLimitedProvider` 装饰器（[rate_limit.go](backend-go/internal/llm/rate_limit.go)），`BuildProviders` 阶段对每个 Provider 透明包裹；环境变量 `RIPPLE_LLM_RPS`（默认 5）+ `RIPPLE_LLM_BURST`（默认 10）；`limiter.Wait(ctx)` 受 ctx cancel 控制。

## P1（半年内必修）

### TD-003 LLM CallRecorder 异步通道有限缓冲
- **现象**：buffer 满（512 条）时丢弃新调用记录，账单不准
- **位置**：[store.PGCallRecorder](backend-go/internal/store/llm_calls.go)
- **建议**：满时改为同步落库 + 告警，宁慢勿丢

### TD-004 Edge `kind=custom` 标签校验仅前端
- **现象**：直接调 `POST /edges` 可绕过前端校验，写入任意字符串
- **位置**：[edge.go](backend-go/internal/service/edge.go)
- **建议**：service 层加白名单 + 标签长度上限（≤32）

### TD-005 InviteService.Accept 非事务
- **现象**：Membership.Upsert 与 Invite.IncrementUseCount 跨表无原子保证；并发接受同 token 可能多算 1 次
- **位置**：[invite.go](backend-go/internal/service/invite.go) `Accept()`
- **建议**：统一进 PG `BEGIN` 事务，或对 invite_id 加 `SELECT FOR UPDATE`

### TD-006 Cloud Worker 重试策略仅记 RetryCount
- **现象**：MarkFailed 直接终态，没有 backoff 重投
- **位置**：[cloud.go](backend-go/internal/service/cloud.go) `process()`
- **建议**：引入 `max_retries` + 指数退避 + 死信队列

### TD-007 Node 修订日志写入用 fmt.Printf
- **现象**：`recordRevision` 失败时通过 `fmt.Printf` 输出 warning，不进 zerolog
- **位置**：[node.go](backend-go/internal/service/node.go)
- **建议**：注入 logger 或返回 error 给上层

### TD-008 Presence 内存模式无并发隔离测试
- **现象**：`memStore.add/remove/list` 各自加锁但跨方法的 read-modify-write 未做压力测试
- **位置**：[presence.go](backend-go/internal/presence/presence.go)
- **建议**：补 -race 多 goroutine 压测

## P2（有空再做）

### TD-009 Claude Code Provider 缺日志脱敏 + 配额识别
- **现象**：claude_code.go 设计文档要求 sha256 prompt 日志、quota exceeded 自动 fallback，未实现
- **位置**：[claude_code.go](backend-go/internal/llm/claude_code.go)（build tag）
- **建议**：启用前完成

### TD-010 前端编辑/历史/邀请均用 prompt/alert 占位
- **现象**：F2/F3 全用浏览器原生弹窗，移动端 UX 差
- **位置**：[Home.tsx](frontend/src/pages/Home.tsx)
- **建议**：M3 阶段引入 Dialog 组件库（保持轻量，自研 Modal 即可）

### TD-011 WS 心跳 ping 不带序列号
- **现象**：客户端 30s 发 `{type:'ping'}`，服务端不返 pong；只用作触发 presence Heartbeat
- **位置**：[wsClient.ts](frontend/src/api/wsClient.ts) / [handlers_ws.go](backend-go/internal/api/http/handlers_ws.go)
- **建议**：服务端可选返 `{type:'pong'}`，前端检测 N 次无响应自动重连

### TD-012 集成测试覆盖率
- **现存**：FullFlow / EdgeFlow / InviteFlow / NodeRevisionFlow / PresenceFlow / AuthGuard / Healthz
- **缺失**：CloudFlow（造云 → 凝露）、WS 协议层（subscribe/edge.create/node.deleted 路由）
- **建议**：M3 启动前补齐

### TD-013 PG 连接池配置无运行时观测
- **现象**：当前只在启动期校验 max>min，运行期连接饱和无指标
- **建议**：Prometheus exporter（M3 监控议题）
