## 二、按层覆盖

### 2.1 单元测试

| 层 | 框架 | 关键覆盖 |
| :--- | :--- | :--- |
| 后端 Python | pytest + pytest-asyncio | Service 函数、Cypher 构造、权限判定 |
| 前端 React | Vitest + Testing Library | 组件渲染、Hook、状态机 |
| AI Prompt | Snapshot test | Prompt 构造结果稳定 |

覆盖率门槛：**Service 层 ≥ 80%**，前端组件 ≥ 70%。

### 2.2 集成测试

| 场景 | 工具 |
| :--- | :--- |
| API + Postgres + Neo4j | pytest + testcontainers |
| WebSocket 协作 | pytest + websockets client |
| 文档炸开流水线 | Celery eager mode + 真实 PDF 样本 |
| Outbox → AI → Notify | 端到端事件链验证 |

### 2.3 E2E

- 框架：Playwright
- 必跑场景（10 个用户故事映射，详见 §六）
- 在 PR 合并前必须通过

### 2.4 视觉回归

- 工具：Chromatic / Percy
- 范围：Delta View 节点、连线、状态切换、动效关键帧
- 阈值：像素差异 ≤ 0.5%

### 2.5 协作并发测试

- 工具：自研 K6 + WS 脚本
- 场景：100 用户同时在 1 湖泊操作
- 验证：LWW 收敛 / 暗流分岔正确生成 / OT 不丢数据

### 2.6 性能与压测

| 测试 | 工具 | 目标 |
| :--- | :--- | :--- |
| API 压测 | k6 / Locust | 满足 G6 SLO |
| Neo4j 大图 | 100 万节点湖泊查询 ≤ 1s | — |
| 移动端离线 | 500 条 Outbox 重放成功率 100% | — |

### 2.7 AI 质量回归

- 数据集：100 条节点对 + 人工标注关系
- 指标：织网命中率 ≥ 75%，幻觉率 ≤ 5%
- 每次 Prompt 调整必须跑回归

## 三、CI/CD 集成

```
PR 提交
  │
  ├─ Lint + 单元测试（必须通过）
  ├─ 集成测试（必须通过）
  ├─ E2E（关键路径子集）
  ├─ 视觉回归（差异需人工确认）
  └─ AI 回归（每周一次完整集）

合并到 main
  │
  └─ 部署到 staging → 全量 E2E + 性能 → 手动批准 → 生产
```

## 四、测试数据

- **种子数据：** `seeds/` 目录提供"西湖示例项目"等可预测样本
- **匿名化：** 生产数据不得直接复制到非生产环境；必须脱敏
- **快照：** Neo4j / PG snapshot 用于复现疑难 bug
