# MVP · 范围、里程碑与评审结论

**版本：** v1.0
**日期：** 2026‑04‑21
**作者：** PM（含 AI 应用专家 + 体验方评审结论）
**用途：** 在进入实现前，框定第一阶段交付边界，避免"什么都想要、什么都做不完"。

---

## 一、MVP 哲学

> **能让一个真实用户独自把"灵感 → 节点 → 引用 → 发布"走通，就是 MVP。**

不是把 D0-D7 + G1-G10 全做完，而是抽出"水文闭环最细的一根线"。

---

## 二、MVP 范围（IN / OUT）

### ✅ IN（必做）

| 模块 | 范围 |
| :--- | :--- |
| 基础底座 | G1 数据层（Lake / Node / Snapshot 三类即可）+ JWT 单设备登录 |
| 造浪池 | D1+D2 单湖单人模式：投石 / 引流 / 蒸发 / 分蘖；最多 500 节点 |
| AI 织网 | G5 单 Agent（仅 Weaver，不做 Critic）+ D7 双节点织网 Prompt |
| 冰山 | G3 一种出口：入海（导出 Markdown）；不做朦胧版/漂流瓶 |
| 文件 | G4 仅支持图片直传 + 单文件 PDF 炸开 |
| onboarding | 教程模式：3 步首次引导 + 节点 hover tooltip 解释术语 |
| 可观测 | G6 基础日志 + Sentry 错误上报；无完整 Grafana 看板 |
| 安全 | G7 §二/三/四 三件套：JWT、HTTPS、节点级 ACL；不做内容审核 |
| 测试 | G8 单元覆盖 ≥ 60% + E2E 跑通故事 1+5 |

### ❌ OUT（V1.1+ 再做）

- 多人实时协作（D5）
- 跨湖运河 / 合流（D6）
- 云霓移动端（G2）
- 朦胧版 / 漂流瓶 / 抛饵（G3 一部分）
- 多 Agent 红蓝军（G5 完整版）
- AI 内容审核（G7 §五）
- i18n 多语言（G10）
- 视觉回归 + 协作压测（G8 一部分）
- WebGL 高级特效（D3 仅做 InstancedMesh 基础版）

## 三、里程碑（不给时间，只给完成度门槛）

### M1 · 骨架贯通（"能跑起来"）
- [ ] 项目脚手架（FastAPI + Next.js + Docker Compose）
- [ ] G1 Schema 落地 + Repository 层
- [ ] JWT 登录 / 注册 / 我的湖泊列表
- [ ] D1 最小画布：拖拽 + 投石 + 节点显示
- **验收：** 能创建账号 → 进入西湖 → 投出第一颗节点

### M2 · AI 流转（"AI 能干活"）
- [ ] G4 图片上传 + 节点附图
- [ ] G5 单 Agent Weaver + Outbox 触发
- [ ] D2 节点连线（虚线/实线）+ AI 暗流可视化
- [ ] D7 双节点织网 Prompt 接入真实 LLM
- **验收：** 投两颗节点 → AI 自动建议关系 → 用户接受变实线

### M3 · 生命周期闭环（"删得掉、找得回、发得出"）
- [ ] G1 节点状态机：DROP/FROZEN/VAPOR
- [ ] 迷雾区 + 30 天 TTL Job
- [ ] G3 决堤口"入海"出口：导出 Markdown
- [ ] 教程模式（onboarding 3 步）
- **验收：** 走通故事 1（自由撰稿人）+ 故事 5（onboarding 林夕）+ 故事 7（误删恢复）

### M4 · 可观测与硬化
- [ ] G6 基础日志 + Sentry
- [ ] G7 节点级 ACL 强约束 + Cypher 自动注入 lake_id
- [ ] G8 E2E 测试 + 单元覆盖门槛
- [ ] 性能验证：500 节点流畅
- **验收：** P99 latency 达标，可以小流量灰度

## 五、体验方评审结论与 MVP 调整

| 痛点 | 严重度 | MVP 是否解决 | 处置 |
| :--- | :---: | :---: | :--- |
| onboarding 缺教程 | 🔴 | ✅ | M3 必交付 3 步教程 + 首颗节点引导 |
| 200+ 节点找不到自己刚投的 | 🟡 | ✅ | MVP 支持"快速定位最新节点"快捷键 |
| 隐喻命名认知负担 | 🟡 | ✅ | 首次出现术语自动 tooltip（hover 1s） |
| 暗流虚线无法操作 | 🔴 | ✅ | M2 必加：虚线 hover → "接受 / 否决"按钮 |
| 迷雾区入口隐蔽 | 🟡 | ✅ | D1 顶部菜单 `[☰]` 增加"迷雾区"项 |
| 离线 500 条无感知 | 🟡 | ❌ | OUT（云霓 V1.1） |
| 协作权重不直观 | 🟡 | ❌ | OUT（D5 V1.1） |
| 决堤口三选一难选 | 🟡 | ✅ | MVP 仅"入海"一个出口，避免选择困难 |

**评审采纳率：** 13 / 16 = **81%**。

## 六、风险与应对

| 风险 | 应对 |
| :--- | :--- |
| AI 调用成本失控 | 单用户 1000 次/天硬限；超过转 fallback |
| WebGL 在低端设备崩 | 检测 GPU 能力，降级为 SVG 简化模式 |
| 用户不理解水文隐喻 | onboarding 教程 + 术语 tooltip + 帮助中心 |
| 节点丢失（最严重） | 每 5min 自动 snapshot；JSON 格式可直接导出兜底 |
| LLM 服务商宕机 | 三态降级链；fallback 模式只生成虚线建议 |

## 七、决策者裁决

> 综合 AI 应用专家与体验方意见，本 MVP 计划：
> - 范围合理（OUT 项明确）
> - 关键风险（幻觉、Prompt 注入、onboarding）已被吸收
> - 里程碑以"完成度门槛"而非时间约束，符合"老板只看结果"的原则
