## 0. Phase 13 推荐启动方式（联调 / 灰度前）

若你要跑的是当前主线的联调环境，不建议再手工拼中间件和环境变量，直接使用项目根的一键脚本：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

$env:PG_PASSWORD="replace-me"
$env:NEO4J_PASSWORD="replace-me"
$env:REDIS_PASSWORD="replace-me"
$env:JWT_SECRET="replace-me-with-at-least-32-chars"

./scripts/bootstrap-staging.ps1
```

默认地址：

- 前端：`http://127.0.0.1:14173`
- 后端：`http://127.0.0.1:18000`

回收环境：

```powershell
./scripts/teardown-staging.ps1
```

若当前机器没有 Docker，请跳到第 1 节走手工模式，或改到联调机执行。


## 2. 启动中间件（docker compose 最快）

项目根新建 `docker-compose.dev.yml`（仅本机用）：

```yaml
services:
  pg:
    image: postgres:16
    environment:
      POSTGRES_USER: ripple
      POSTGRES_PASSWORD: ripple
      POSTGRES_DB: ripple
    ports: ["5432:5432"]

  neo4j:
    image: neo4j:5
    environment:
      NEO4J_AUTH: neo4j/ripple-pass
    ports: ["7474:7474", "7687:7687"]

  redis:
    image: redis:7
    command: redis-server --requirepass ripple-pass
    ports: ["6379:6379"]
```

```powershell
docker compose -f docker-compose.dev.yml up -d
```

> 已有现成中间件可跳过本节，按实际地址填到下面的 `.env`。

## 3. 后端：跑迁移 + 起服务

```powershell
cd backend-go

# 3.1 配置环境变量
copy .env.example .env
# 用编辑器打开 .env，至少填以下几项（PG/Neo4j/Redis 用第 2 节的地址）：
#   RIPPLE_PG_URL=postgres://ripple:ripple@localhost:5432/ripple?sslmode=disable
#   RIPPLE_NEO4J_URI=bolt://localhost:7687
#   RIPPLE_NEO4J_USER=neo4j
#   RIPPLE_NEO4J_PASSWORD=ripple-pass
#   RIPPLE_REDIS_ADDR=localhost:6379
#   RIPPLE_REDIS_PASSWORD=ripple-pass
#   RIPPLE_JWT_SECRET=<随机 32 字节字符串>

# 3.2 跑 PG 迁移（按编号顺序）
$env:PGPASSWORD="ripple"
foreach ($f in Get-ChildItem migrations\*.up.sql | Sort-Object Name) {
    psql -h localhost -U ripple -d ripple -f $f.FullName
}

# 3.3 启动服务（默认监听 :8080）
go run ./cmd/server
```

启动日志若出现 `claude code cli not available`，是因为本机未装 Claude Code CLI；属正常 warn，不影响其它 provider。要启用：
```ini
RIPPLE_CLAUDE_CODE_ENABLED=true
RIPPLE_CLAUDE_CODE_CLI_PATH=    # 留空走 PATH；Windows 一般是 npm 全局目录的 claude.cmd
```

健康检查：
```powershell
curl http://localhost:8080/healthz
# {"status":"ok"}
```

## 5. 第一次跑通：5 步看到 AI 造云

1. **注册**：右上角注册账号（邮箱 + 密码 + 显示名）
2. **建湖**：左侧侧栏点"+"创建一个 Lake，例如 "我的灵感池"
3. **写节点**：在湖内输入框输入一段灵感（默认创建 MIST 状态节点）
4. **触发造云**：点节点旁的 ☁ 按钮，等 1-3 秒，CLOUD 节点会自动出现并连边
5. **协作**：右上角"邀请"生成链接，浏览器隐身窗口打开 → 注册另一个账号 → 粘贴链接接受 → 两个窗口同时看到 Presence 头像

> 若造云失败：检查 `.env` 是否配了至少一个 LLM provider key（`RIPPLE_ZHIPU_API_KEY` / `RIPPLE_DEEPSEEK_API_KEY` / `RIPPLE_OPENAI_API_KEY` / 或本机 Ollama 的 `RIPPLE_OPENAI_COMPAT_*`）。

## 8. 卡住了？

| 现象 | 排查 |
|------|------|
| 后端起不来：`pg connect failed` | 检查 docker 服务是否健康；`.env` 的连接串拼写 |
| Neo4j 连不上 | 5+ 默认 `neo4j` 库；`RIPPLE_NEO4J_DATABASE=neo4j`（一般不用改） |
| Redis 鉴权失败 | 检查 docker compose 里 `--requirepass` 是否与 `.env` 一致 |
| 前端空白 | F12 看 Console；多半是 401（token 失效），重登 |
| 造云一直 `pending` | 后端日志看 weaver；八成 LLM provider key 错或网络出不去 |
| Windows 中文显示乱码 | PowerShell GBK 编码问题；文件本身是 UTF-8。`chcp 65001` 后再看 |
