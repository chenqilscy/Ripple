## 官网 / 公众号长版

今天，我们正式发布青萍 Ripple v0.14.0。

如果说过去的 Ripple 更像一片正在聚拢水势的湖面，那么从 v0.14.0 开始，它第一次具备了真正的“运营能力”：

- 组织级配额不再停留在表结构，而是进入了建湖、成员、节点、附件、API Key 的全链路拦截；
- 平台管理员终于拥有了一块能看全局、管灰度、做准入的运营面板；
- 灰度名单、平台管理员角色、准入脚本、三签字机制全部闭环，发布不再依赖“口头确认”；
- 本地脚本化质量门替代云端 CI 绿线，准入过程更可控，也更贴近真实环境。

这不是一个“新按钮集合”的版本，而是 Ripple 从“功能系统”进入“产品系统”的关键一步。

我们一直相信，创意不该被钢筋水泥式的软件框住。风起于青萍之末，浪成于微澜之间。真正好的创作工具，不只是让你把内容写出来，还要让团队知道如何管理它、保护它、放大它、把它安全地带到更远的地方。

v0.14.0，就是这一步。

## 本次重点发布

### 1. 组织配额全链路上线

- 组织默认免费套餐正式生效；
- 湖、成员、节点、附件、API Key 全部按组织配额计量；
- 超限路径明确返回业务错误，不再出现“静默失败”；
- Condense 同组织内防重计逻辑落地，避免 AI 生成路径重复消耗配额。

### 2. 运营管理台进入可用态

- Settings 内新增管理员总览；
- 组织列表直接展示 Members / Lakes / Nodes 用量摘要；
- 最近配额审计记录可见，方便追溯谁在什么时候改了什么。

### 3. 灰度名单与平台管理员体系闭环

- 灰度名单支持后台 CRUD；
- 平台管理员列表支持新增、撤销、OWNER 二次确认；
- 内测入口、权限边界、实际操作路径第一次完整打通。

### 4. 发布准入流程标准化

- `go test -race -count=1 ./...`、`go vet ./...`、前端 lint / test / build、Playwright E2E、本地 acceptance 脚本全部纳入准入门；
- staging smoke 数据支持 dry-run 与真实清理；
- PM / QA / 决策者三签字正式成为发布前置条件。

## 对谁有价值

### 对创作者

你不必再担心“团队一多，湖一多，系统会不会乱”。配额和权限开始真正保护创作秩序。

### 对团队 OWNER / ADMIN

你第一次可以在一个地方看到组织用量、灰度入口、管理员配置和准入证据链，而不是在聊天记录里翻日志。

### 对平台运营者

你终于拥有一套可执行的发布闭环：能回归、能验收、能签字、能回溯。

## 当前边界

v0.14.0 解决的是“可运营、可管理、可准入”，而不是终点。以下内容已经进入下一阶段：

- AI Workflow 扩展（AI 节点、Prompt 模板库、多节点整理）；
- 商业化与计量（套餐升级、AI 用量账单、订阅路径）；
- 移动端与更完整的离线写能力。

换句话说，v0.14.0 是把水路修通；v0.15.0 开始，我们会让更多浪真正涌起来。
