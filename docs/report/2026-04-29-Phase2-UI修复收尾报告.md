## 二、本轮新增（Phase 2 收尾）

### P2-03 边 hover 深度实现
- **后端**：`domain.Edge.Strength float64`，Neo4j Cypher 加 `coalesce(e.strength, 0.0)`，API 响应加 `strength,omitempty`，`scanEdge` 读 vals[9]
- **前端**：`EdgeItem.strength?`，`SimLink.strength/kind`，`SpringEdges` 内 `gl.domElement mousemove` 监听，Three.js 世界坐标反投影 + 逐边最小距离，命中阈值时 callback 传出坐标/强度/kind
- **Tooltip**：LakeGraph 叠加 DOM tooltip，显示 `kind` + `相似度 XX%`（strength>0 时）
- **单测**：`TestEdge_Strength_PersistedAndReturned` + `TestScanEdge_Strength`（4 subcase），全部 PASS

### SW 更新策略
- `install` 不再自动 `skipWaiting()`
- 页面检测到新 SW `installed` 状态时，底部弹出 Toast："🎯 新版本已就绪"+"立即刷新"按钮+"✕ 关闭"
- 点击刷新：`postMessage({type:'SKIP_WAITING'})` → SW 接管 → 页面刷新
- SW 监听 `message` 事件处理 `SKIP_WAITING`

### P1-04 步骤式引导卡片
- 空湖节点数为 0 时，显示三步引导卡片：
  - ① 添加节点：手工输入知识点
  - ② 造云 · AI 发散：一键生成相关节点
  - ③ 建立关联：连线建立知识关系
- 青碧色边框（`rgba(46,139,144,0.2)`），与品牌色一致
