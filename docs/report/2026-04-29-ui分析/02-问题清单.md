## 二、P0 问题清单（立即修复）

### P0-01：错误提示暴露技术细节

**问题描述：** 所有截图右下角显示红色Toast："LLM error: zhipu: does not support streaming"

**证据锚点：** S1/S3/S4/S5/S6右下角红色Toast

**影响范围：** 所有用户，所有页面

**优先级理由：** 
- 违背品牌"水文隐喻"设计理念
- 暴露技术实现细节（zhipu、streaming）
- 降低用户信任度

**修复建议：**
1. 前端：将技术错误信息映射为品牌化文案
   - `LLM error` → `潮汐异常`
   - `does not support streaming` → `AI暂歇，请稍候`
2. 后端：将详细错误信息写入日志系统（Sentry/CloudWatch）
3. 设计：错误提示使用品牌色（青碧#2E8B57）而非红色

**预估工时：** 4小时（前端2h + 后端1h + 测试1h）

**验收标准：**
- [ ] 所有错误提示不再包含技术术语（LLM、zhipu、streaming、API等）
- [ ] 错误信息已写入后端日志系统，包含完整堆栈
- [ ] 设计师确认文案符合品牌调性

**实施阶段：** Phase 1

---

### P0-02：节点卡片操作按钮过多

**问题描述：** 每个节点卡片显示12个操作按钮，没有主次之分，视觉混乱

**证据锚点：** S5，每个卡片12个按钮（眼睛、刷新、下载、复制、编辑、删除、分享、标签、附件、评论、历史、更多）

**影响范围：** 列表视图，所有节点卡片

**优先级理由：**
- 违背"呼吸感"设计原则（大量留白）
- 增加认知负担，用户难以快速定位主要操作
- 移动端几乎不可用

**修复建议：**
1. 主要操作（3个）：始终可见
   - 查看详情（眼睛图标）
   - 编辑（铅笔图标）
   - 更多（三点图标，展开菜单）
2. 次要操作（9个）：收入"更多"菜单
   - 刷新、下载、复制、删除、分享、标签、附件、评论、历史
3. 悬停态：显示所有按钮（桌面端）

**预估工时：** 6小时（设计2h + 前端3h + 测试1h）

**验收标准：**
- [ ] 默认状态仅显示3个按钮
- [ ] "更多"菜单包含9个次要操作
- [ ] 桌面端悬停显示所有按钮
- [ ] 移动端点击"更多"展开菜单
- [ ] 设计师确认符合"呼吸感"原则

**实施阶段：** Phase 1

### P0-03：图标缺少Tooltip

**问题描述：** 所有图标按钮无文字说明，用户需要逐个点击才能理解功能

**证据锚点：** S5，所有图标按钮，悬停1秒无Tooltip

**影响范围：** 所有图标按钮（节点卡片、工具栏、导航栏）

**优先级理由：**
- 学习成本高，新用户不理解图标含义
- 部分图标语义不清（眼睛=查看？预览？）
- 违背可访问性标准（WCAG 2.1 AA）

**修复建议：**
1. 所有图标按钮添加Tooltip（悬停0.5秒显示）
2. Tooltip文案使用品牌术语：
   - 删除 → "蒸发（删除）"
   - 复制 → "分蘖（复制）"
   - 保存 → "固形（保存）"
3. 移动端：长按显示Tooltip

**预估工时：** 3小时（前端2h + 测试1h）

**验收标准：**
- [ ] 所有图标按钮有Tooltip
- [ ] Tooltip在悬停0.5秒后显示
- [ ] Tooltip文案包含品牌术语和通用词汇
- [ ] 移动端长按显示Tooltip
- [ ] 屏幕阅读器能读取Tooltip内容

**实施阶段：** Phase 1

## 四、P2 问题清单（长期规划）

### P2-01：图谱渲染性能风险

**问题描述：** 65个节点已拥挤，扩展到500+节点可能卡顿

**证据锚点：** S1，65个节点密集重叠

**影响范围：** 图谱视图

**优先级理由：**
- 当前可用，但扩展性差
- 需要架构级优化

**修复建议：**
1. 使用Canvas/WebGL渲染（替代SVG/DOM）
2. 实现LOD（Level of Detail）：
   - 缩小时：显示聚合节点
   - 放大时：显示详细节点
3. 虚拟化：仅渲染可见区域节点
4. Web Worker：后台计算布局

**预估工时：** 40小时（架构设计8h + 前端28h + 测试4h）

**验收标准：**
- [ ] 支持500+节点流畅渲染（60fps）
- [ ] 缩放/平移流畅
- [ ] 内存占用<500MB

**实施阶段：** Phase 3
