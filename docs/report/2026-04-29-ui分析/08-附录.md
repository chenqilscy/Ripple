## 五、测试用例

### 5.1 功能测试用例

#### TC-01：创建节点

**前置条件：** 用户已登录，进入湖泊

**步骤：**
1. 点击"投石"按钮
2. 输入节点标题
3. 点击"固形"按钮

**预期结果：**
- 节点出现在图谱中
- Toast提示"节点已固形"
- 节点状态为MIST

---

#### TC-02：删除节点

**前置条件：** 用户已登录，湖泊中有节点

**步骤：**
1. 选择节点
2. 点击"删除（蒸发）"按钮
3. 确认删除

**预期结果：**
- 节点从图谱中消失
- Toast提示"节点已蒸发"
- 节点状态变为VAPOR

---

#### TC-03：批量删除节点

**前置条件：** 用户已登录，湖泊中有多个节点

**步骤：**
1. 勾选多个节点
2. 点击批量操作栏"删除（蒸发）"按钮
3. 确认删除

**预期结果：**
- 所有选中节点从图谱中消失
- Toast提示"已蒸发N个节点"

---

### 5.2 可访问性测试用例

#### TC-A01：键盘导航

**步骤：**
1. 使用Tab键遍历所有交互元素
2. 使用Enter/Space激活按钮
3. 使用Esc关闭弹窗

**预期结果：**
- 所有交互元素可通过Tab键访问
- 焦点样式明显
- 无键盘陷阱

---

#### TC-A02：屏幕阅读器

**工具：** NVDA（Windows）或VoiceOver（macOS）

**步骤：**
1. 启动屏幕阅读器
2. 导航到节点卡片
3. 激活"删除（蒸发）"按钮

**预期结果：**
- 屏幕阅读器读出"删除（蒸发）节点"
- 操作成功后读出"节点已蒸发"

---

### 5.3 性能测试用例

#### TC-P01：图谱渲染性能

**步骤：**
1. 创建500个节点
2. 打开图谱视图
3. 使用Performance API测量渲染时间

**预期结果：**
- 渲染时间<2s
- 渲染帧率>60fps

---

#### TC-P02：缩放性能

**步骤：**
1. 打开包含500个节点的图谱
2. 使用鼠标滚轮缩放10次
3. 使用Performance API测量帧率

**预期结果：**
- 缩放流畅，无卡顿
- 帧率>60fps

## 六、术语表

| 术语 | 英文 | 含义 |
|------|------|------|
| **湖泊** | Lake | 知识的容器，类似"项目"或"工作区" |
| **节点** | Node | 知识单元，可以是想法、文档、链接等 |
| **关联** | Edge | 节点之间的连接，表示知识关系 |
| **投石** | Drop | 创建新节点 |
| **蒸发** | Evaporate | 删除节点 |
| **固形** | Solidify | 保存节点 |
| **分蘖** | Tiller | 复制节点 |
| **引流** | Channel | 创建关联 |
| **造浪池** | Wave Pool | 图谱视图，核心交互界面 |
| **航行者** | Navigator | 用户 |
| **舟子** | Owner | 湖泊所有者 |
| **副舟子** | Navigator | 湖泊编辑者 |
| **渡客** | Passenger | 湖泊评论者 |
| **观潮** | Observer | 湖泊观察者 |
