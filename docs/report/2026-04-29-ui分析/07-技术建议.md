## 一、前端架构建议

### 1.1 图谱渲染优化（P2-01）

#### 当前问题

**证据：** S1，65个节点已密集重叠

**性能风险：**
- 当前使用SVG/DOM渲染，节点>200时可能卡顿
- 力导向布局算法复杂度O(n²)，节点增多时计算缓慢
- 无虚拟化，所有节点始终在DOM中

---

#### 技术方案

##### 方案1：Canvas渲染（推荐）

**适用场景：** 200-1000个节点

**优点：**
- 渲染性能高（60fps）
- 内存占用低
- 支持大规模节点

**缺点：**
- 交互逻辑需手动实现（点击、悬停、拖拽）
- 无法使用CSS样式
- 可访问性需额外处理

**实现建议：**
```javascript
// 使用PixiJS或Konva.js
import { Application, Graphics } from 'pixi.js';

const app = new Application({
  width: window.innerWidth,
  height: window.innerHeight,
  antialias: true,
  backgroundColor: 0x1A1A1A
});

// 渲染节点
nodes.forEach(node => {
  const circle = new Graphics();
  circle.beginFill(0x2E8B57);
  circle.drawCircle(node.x, node.y, node.radius);
  circle.endFill();
  circle.interactive = true;
  circle.on('pointerdown', () => handleNodeClick(node));
  app.stage.addChild(circle);
});
```

**工时：** 20小时

---

##### 方案2：WebGL渲染（高性能）

**适用场景：** 1000+个节点

**优点：**
- 极高性能（支持10000+节点）
- GPU加速
- 支持复杂视觉效果

**缺点：**
- 实现复杂度高
- 浏览器兼容性（需降级方案）
- 调试困难

**实现建议：**
```javascript
// 使用Three.js或Sigma.js
import Sigma from 'sigma';
import Graph from 'graphology';

const graph = new Graph();
nodes.forEach(node => graph.addNode(node.id, node));
edges.forEach(edge => graph.addEdge(edge.source, edge.target, edge));

const renderer = new Sigma(graph, container, {
  renderEdgeLabels: false,
  enableEdgeEvents: true
});
```

**工时：** 28小时

---

##### 方案3：混合渲染（平衡）

**适用场景：** 需要兼顾性能和可访问性

**策略：**
- 节点<100：SVG/DOM渲染（保持可访问性）
- 节点100-500：Canvas渲染
- 节点>500：WebGL渲染 + LOD（Level of Detail）

**实现建议：**
```javascript
function selectRenderer(nodeCount) {
  if (nodeCount < 100) return new SVGRenderer();
  if (nodeCount < 500) return new CanvasRenderer();
  return new WebGLRenderer();
}

const renderer = selectRenderer(nodes.length);
renderer.render(nodes, edges);
```

**工时：** 40小时

---

#### LOD（Level of Detail）策略

**目标：** 缩小时显示聚合节点，放大时显示详细节点

**实现：**
```javascript
function getLOD(zoomLevel) {
  if (zoomLevel < 0.5) {
    // 显示聚合节点（按类型/标签分组）
    return aggregateNodes(nodes);
  } else if (zoomLevel < 1.5) {
    // 显示简化节点（仅圆形+标题）
    return simplifyNodes(nodes);
  } else {
    // 显示完整节点（圆形+标题+图标+标签）
    return nodes;
  }
}
```

**工时：** 8小时

---

#### 虚拟化策略

**目标：** 仅渲染可见区域节点

**实现：**
```javascript
function getVisibleNodes(nodes, viewport) {
  return nodes.filter(node => {
    return node.x >= viewport.x &&
           node.x <= viewport.x + viewport.width &&
           node.y >= viewport.y &&
           node.y <= viewport.y + viewport.height;
  });
}

// 仅渲染可见节点
const visibleNodes = getVisibleNodes(nodes, viewport);
renderer.render(visibleNodes, edges);
```

**工时：** 6小时

---

#### 布局算法优化

**当前问题：** 力导向布局O(n²)复杂度

**优化方案：**

1. **使用Web Worker后台计算**
```javascript
// main.js
const worker = new Worker('layout-worker.js');
worker.postMessage({ nodes, edges });
worker.onmessage = (e) => {
  updateNodePositions(e.data.nodes);
};

// layout-worker.js
self.onmessage = (e) => {
  const { nodes, edges } = e.data;
  const positions = forceDirectedLayout(nodes, edges);
  self.postMessage({ nodes: positions });
};
```

2. **使用Barnes-Hut算法降低复杂度**
   - 复杂度：O(n²) → O(n log n)
   - 库：d3-force（内置Barnes-Hut）

3. **布局降采样**
   - 每10帧计算一次布局（而非每帧）
   - 中间帧使用插值

**工时：** 8小时

### 1.2 状态管理优化

#### 当前问题

**推测：** 可能使用React Context或Redux，大规模数据时可能性能问题

---

#### 建议方案

##### 方案1：使用Zustand（轻量级）

**优点：**
- 简单易用
- 性能好（基于订阅）
- 无需Provider

**示例：**
```javascript
import create from 'zustand';

const useStore = create((set) => ({
  nodes: [],
  edges: [],
  selectedNode: null,
  addNode: (node) => set((state) => ({ 
    nodes: [...state.nodes, node] 
  })),
  selectNode: (id) => set({ selectedNode: id })
}));

// 使用
const nodes = useStore((state) => state.nodes);
const addNode = useStore((state) => state.addNode);
```

**工时：** 4小时（迁移现有状态）

---

##### 方案2：使用Jotai（原子化）

**优点：**
- 细粒度更新（仅更新使用的组件）
- 适合大规模数据
- TypeScript支持好

**示例：**
```javascript
import { atom, useAtom } from 'jotai';

const nodesAtom = atom([]);
const selectedNodeAtom = atom(null);

// 使用
const [nodes, setNodes] = useAtom(nodesAtom);
const [selectedNode, setSelectedNode] = useAtom(selectedNodeAtom);
```

**工时：** 6小时（迁移现有状态）

### 1.3 实时协作优化（P2-02）

#### 技术方案

##### 方案1：WebSocket + CRDT

**适用场景：** 多人实时协作编辑

**架构：**
```
客户端A ←→ WebSocket ←→ 服务器 ←→ WebSocket ←→ 客户端B
              ↓                        ↓
           CRDT同步                 CRDT同步
```

**CRDT库推荐：**
- Yjs（推荐）：成熟、性能好、支持多种数据类型
- Automerge：功能强大，但性能较低

**示例（Yjs）：**
```javascript
import * as Y from 'yjs';
import { WebsocketProvider } from 'y-websocket';

const ydoc = new Y.Doc();
const nodes = ydoc.getArray('nodes');
const provider = new WebsocketProvider('ws://localhost:1234', 'room-id', ydoc);

// 监听变化
nodes.observe((event) => {
  console.log('节点变化:', event.changes);
  updateUI();
});

// 添加节点
nodes.push([{ id: '1', title: '新节点' }]);
```

**工时：** 24小时（前端16h + 后端8h）

##### 方案2：WebSocket + OT（Operational Transformation）

### 1.4 错误处理优化（P0-01）

#### 当前问题

**证据：** S1/S3/S4/S5/S6，错误提示"LLM error: zhipu: does not support streaming"

**问题：**
- 暴露技术细节
- 无错误分类
- 无错误日志

---

#### 技术方案

##### 1. 错误分类和映射

**实现：**
```javascript
// 错误映射表
const errorMessages = {
  'LLM_ERROR': '潮汐异常，AI暂歇',
  'NETWORK_ERROR': '水流中断，请检查网络',
  'PERMISSION_DENIED': '航行受限，请联系管理员',
  'RATE_LIMIT': '潮汐频繁，请稍后再试'
};

// 错误处理函数
function handleError(error) {
  const errorType = classifyError(error);
  const userMessage = errorMessages[errorType] || '未知异常，请联系支持';
  
  // 显示用户友好的错误提示
  showToast(userMessage, 'error');
  
  // 记录详细错误到日志系统
  logError(error, { errorType, userMessage });
}

// 错误分类
function classifyError(error) {
  if (error.message.includes('LLM')) return 'LLM_ERROR';
  if (error.message.includes('Network')) return 'NETWORK_ERROR';
  if (error.status === 403) return 'PERMISSION_DENIED';
  if (error.status === 429) return 'RATE_LIMIT';
  return 'UNKNOWN_ERROR';
}
```

**工时：** 2小时

---

##### 2. 错误日志系统

**推荐：** Sentry（前端错误监控）

**集成：**
```javascript
import * as Sentry from '@sentry/react';

Sentry.init({
  dsn: 'YOUR_SENTRY_DSN',
  environment: process.env.NODE_ENV,
  beforeSend(event, hint) {
    // 脱敏：移除敏感信息
    if (event.request) {
      delete event.request.cookies;
      delete event.request.headers['Authorization'];
    }
    return event;
  }
});

// 记录错误
function logError(error, context) {
  Sentry.captureException(error, {
    tags: { errorType: context.errorType },
    extra: { userMessage: context.userMessage }
  });
}
```

**工时：** 1小时

---

##### 3. 错误恢复策略

**实现：**
```javascript
// 自动重试（指数退避）
async function fetchWithRetry(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fetch(url, options);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await sleep(Math.pow(2, i) * 1000); // 1s, 2s, 4s
    }
  }
}

// 降级方案
async function fetchNodes() {
  try {
    return await fetchFromAPI('/nodes');
  } catch (error) {
    // 降级：从本地缓存读取
    return await fetchFromCache('/nodes');
  }
}
```

**工时：** 4小时

**优点：**
- 客户端按需查询（避免over-fetching）

```graphql
      title

    members {

      name
      role
    }

}
```

### 2.2 实时协作后端

#### 技术选型

**推荐：** Socket.IO + Redis

**架构：**
```
客户端 ←→ Socket.IO ←→ Node.js服务器 ←→ Redis Pub/Sub
                                      ↓
                                   数据库
```

**实现：**
```javascript
// server.js
const io = require('socket.io')(server);
const redis = require('redis');
const subscriber = redis.createClient();
const publisher = redis.createClient();

io.on('connection', (socket) => {
  socket.on('join-lake', (lakeId) => {
    socket.join(lakeId);
    subscriber.subscribe(`lake:${lakeId}`);
  });
  
  socket.on('update-node', (data) => {
    publisher.publish(`lake:${data.lakeId}`, JSON.stringify(data));
  });
});

subscriber.on('message', (channel, message) => {
  const data = JSON.parse(message);
  io.to(data.lakeId).emit('node-updated', data);
});
```

**工时：** 16小时

## 四、安全建议

### 4.1 前端安全

#### 1. XSS防护

**风险：** 节点标题/内容可能包含恶意脚本

**防护：**
```javascript
// 使用DOMPurify清理HTML
import DOMPurify from 'dompurify';

function renderNodeTitle(title) {
  const clean = DOMPurify.sanitize(title);
  return <div dangerouslySetInnerHTML={{ __html: clean }} />;
}
```

---

#### 2. CSRF防护

**防护：**
```javascript
// 所有POST/PUT/DELETE请求携带CSRF token
axios.defaults.headers.common['X-CSRF-Token'] = getCsrfToken();
```

---

### 4.2 后端安全

#### 1. 权限验证

**建议：** 每个API请求验证用户权限

```javascript
// 中间件
function checkPermission(requiredRole) {
  return (req, res, next) => {
    if (req.user.role < requiredRole) {
      return res.status(403).json({ error: 'PERMISSION_DENIED' });
    }
    next();
  };
}

// 使用
app.delete('/api/nodes/:id', checkPermission('NAVIGATOR'), deleteNode);
```

---

#### 2. 速率限制

**建议：** 防止API滥用

```javascript
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 100 // 最多100个请求
});

app.use('/api/', limiter);
```
