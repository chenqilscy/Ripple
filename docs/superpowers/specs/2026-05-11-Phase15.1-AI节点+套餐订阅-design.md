## 一、设计决策汇总

| 决策点 | 选择 | 理由 |
|--------|------|------|
| C.1 异步模式 | 混合模式（轮询3次1s/2s/3s → 10s长轮询） | 兼顾快速响应和低开销 |
| C.1 输入方式 | 即时选节点触发 | 符合"选中节点→AI整理"核心场景 |
| C.1 失败重试 | 自动重试1次 | 与MVP决策一致，用户体验好 |
| D.1 套餐配置 | 前端写死（免费/专业/团队） | Phase 15 MVP够用，修改需发版可接受 |
| D.1 支付流程 | 中间页模拟（Stub支付） | 完整支付体验，便于后续扩展真实网关 |

## 二、C.1 AI节点 API

### 2.1 核心流程

```
用户选中节点 → 点击"AI整理" → 前端POST /ai_trigger
→ 后端创建ai_generated节点 → 返回node_id
→ 前端轮询GET /ai_status → 状态done → 显示结果
```

### 2.2 新增端点

#### POST /api/v1/lakes/{lake_id}/ai-nodes

**创建AI节点（即时选节点触发）**

> 注：路径从 `/nodes/ai_trigger` 改为 `/ai-nodes`，更符合RESTful风格

Request:
```json
{
  "source_node_ids": ["node_uuid_1", "node_uuid_2"],
  "prompt_template_id": "template_uuid",
  "user_intent": "帮我整理成学习笔记",
  "idempotency_key": "client-uuid-xxx"
}
```

| 字段 | 类型 | 必填 | 限制 | 说明 |
|------|------|------|------|------|
| source_node_ids | []string | ✅ | 1-20个 | 用户选中的节点ID，至少1个，至多20个 |
| prompt_template_id | string | ✅ | - | Prompt模板ID |
| user_intent | string | ❌ | ≤500字符 | 用户意图描述，会注入到Prompt |
| idempotency_key | string | ❌ | UUID格式 | 幂等键，60分钟内相同key返回409 |

Response (202 Accepted):
```json
{
  "ai_node_id": "ai_node_uuid",
  "status": "pending",
  "source_count": 2,
  "created_at": "2026-05-11T10:00:00Z"
}
```

错误码：
- 400：source_node_ids为空或无效（<1个或>20个）
- 403：source_node存在但用户无读取权限
- 404：source_node或prompt_template不存在
- 409：idempotency_key重复（60分钟内相同key返回已创建的ai_node_id）
- 422：超出ai_calls_per_month配额限制

#### GET /api/v1/lakes/{lake_id}/ai-nodes/{ai_node_id}/status

**轮询AI节点状态（混合模式）**

> 注：路径改为 `/ai-nodes/{id}/status`，与创建端点一致

Response:
```json
{
  "ai_node_id": "ai_node_uuid",
  "status": "pending|processing|done|failed",
  "progress": 0.5,
  "content": "...",
  "error": "...",
  "retry_count": 0
}
```

| status | 说明 |
|--------|------|
| pending | 等待处理 |
| processing | AI正在生成 |
| done | 完成，content已填充 |
| failed | 失败，error包含原因 |

混合模式实现：
1. 前端先轮询 3 次（间隔 1s → 2s → 3s）
2. 若未完成，降级为 10s 长轮询
3. 超过 5 分钟未完成，返回 timeout 错误

### 2.4 后端流程（Weaver Worker）

```
POST /ai-nodes API
    ↓
检查idempotency_key（60分钟内相同key返回409）
    ↓
验证source_node_ids权限（全部属于当前lake + 用户有读权限）
    ↓
验证prompt_template_id存在且用户有权限
    ↓
校验ai_calls_per_month配额
    ↓
创建ai_generated节点（status=pending）
    ↓
写入Outbox事件ai_node_created（与节点创建在同一事务）
    ↓
返回202 Accepted
    ↓
Weaver Worker消费事件
    ↓
读取source_node内容 → 注入Prompt模板
    ↓
调用LLM（带30s超时，超过则直接失败不重试）
    ↓
更新节点content和ai_status
```

**幂等性实现：**
```go
func (h *AINodeHandler) CreateAINode(c *gin.Context) {
    var req CreateAINodeRequest
    // ...
    idempotencyKey := req.IdempotencyKey

    if idempotencyKey != "" {
        // 检查60分钟内是否有相同key的请求
        existing, err := h.checkIdempotencyKey(lakeID, idempotencyKey)
        if err != nil {
            return err
        }
        if existing != nil {
            // 返回已创建的节点
            c.JSON(409, gin.H{
                "error": "duplicate_request",
                "existing_ai_node_id": existing.ID,
                "message": "该请求已在60分钟内处理过"
            })
            return
        }
    }
    // 继续创建逻辑...
}
```

**Prompt注入防护：**
```go
func buildPromptContext(sourceNodes []Node, template PromptTemplate, userIntent string) string {
    var sb strings.Builder

    // 用户意图
    if userIntent != "" {
        sb.WriteString("用户意图：")
        sb.WriteString(userIntent)
        sb.WriteString("\n\n")
    }

    // 使用sentinel隔离用户内容，防止Prompt注入
    sb.WriteString("###USER_CONTENT_START###\n")
    for i, node := range sourceNodes {
        sb.WriteString(fmt.Sprintf("[节点%d]\n%s\n---\n", i+1, node.Content))
    }
    sb.WriteString("###USER_CONTENT_END###\n")

    return sb.String()
}
```

### 2.5 安全考虑

- source_node_ids 必须全部属于当前 lake（lake_id 过滤）
- source_node_ids 必须用户有读权限，无权限时返回 403
- prompt_template_id 必须属于当前用户或 Org（数据隔离）
- **Prompt注入防护**：source_node内容注入时使用 sentinel 包裹（`###USER_CONTENT_START###` / `###USER_CONTENT_END###`）
- AI输出写入前做内容长度校验（max 10000 chars）
- Token 预算控制（已在 Phase 14 的 org_quotas 限制）

**API限流：**
- 每用户 ≤10次 AI节点创建/分钟
- 超出限流返回 429 Too Many Requests

**配额计数：**
- 触发即计入 `ai_calls_per_month` 配额（无论成功失败）
- 失败重试不额外计费（同一请求内的重试）

### 3.3 数据模型

```sql

CREATE TABLE ai_node_idempotency_keys (

-- org_subscriptions 表（新）
CREATE TABLE org_subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id UUID NOT NULL REFERENCES organizations(id),

    status VARCHAR(20) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'cancelled', 'expired')),
```

## 四、错误处理汇总

| 端点 | 错误码 | 场景 |
|------|--------|------|
| POST /ai-nodes | 400 | source_node_ids为空（<1）或超限（>20） |
| POST /ai-nodes | 403 | source_node存在但用户无读取权限 |
| POST /ai-nodes | 404 | source_node或template不存在 |
| POST /ai-nodes | 409 | idempotency_key重复（60分钟内） |
| POST /ai-nodes | 422 | 超出ai_calls_per_month配额 |
| POST /ai-nodes | 429 | 超出限流（每用户≤10次/分钟） |
| GET /ai-nodes/{id}/status | 404 | ai_node不存在 |
| GET /subscription | 404 | 组织无订阅记录 |
| POST /subscription/preview | 400 | target_plan无效 |
| POST /subscription | 400 | order_id不匹配 |
| POST /subscription | 422 | 降级超出用量 |
| POST /subscription | 409 | 重复订阅同一套餐 |
