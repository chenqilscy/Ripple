# Phase 14 老板体验试用实操指南

> 阅读对象：老板（产品第一体验方）  
> 目标：在 staging 环境 `http://fn.cky:14173` 完整走一遍 Phase 14 新功能，按「体验反馈模板」填写反馈。  
> 预计时间：20-30 分钟  
> 前置：浏览器能访问 `http://fn.cky:14173`；有平台管理员账号（或请 AI 提前创建一个测试账号）

---

## Step 0 · 准备账号

### 方案 A：使用已有平台管理员账号

如果已有账号，直接跳到 Step 1。

### 方案 B：AI 帮你注册一个体验账号

```
灰度名单中添加你的邮箱（需先由现有平台管理员操作）：
  Settings → 灰度名单 → 新增 → 输入你的邮箱 → 保存

然后访问 http://fn.cky:14173 → 注册
```

## Step 1 · 登录并查看造浪池主界面

1. 访问 `http://fn.cky:14173`
2. 登录（邮箱 + 密码）
3. 进入湖列表（主页）
4. **体验要点**：
   - 操作是否流畅？
   - 湖列表加载速度如何？
   - Three.js 粒子效果是否正常渲染？

> 📝 记录你的感受（填入反馈表：实际表现 / 期望表现）

---

## Step 2 · Settings 页面 5 个 Tab 试用

点击右上角头像 / 齿轮图标 → 进入 Settings。

### 2.1 账户设置 Tab

- 确认账户信息展示正常

### 2.2 API Key Tab

- 点击"创建 API Key"，输入名称，提交
- 验证 Key 列表中出现新建的 Key
- 删除刚创建的 Key
- **体验要点**：操作是否有反馈？Key 显示格式是否友好？

### 2.3 灰度名单 Tab（平台管理员可见）

- 点击"添加邮箱"，输入一个测试邮箱（如 `test_experience@ripple.test`），填备注
- 点击保存，确认出现在列表中
- 点击删除，确认弹出二次确认弹窗，点击确认，确认已从列表移除
- **体验要点**：弹窗文案是否清晰？操作是否符合预期？

### 2.4 平台管理员 Tab（平台管理员可见）

- 查看当前管理员列表
- （如有第二个测试账号）点击添加，填入邮箱，保存，确认出现
- 如果刚添加了测试账号，点击撤销按钮，在弹窗中先点"取消"（记录保持），再点撤销，在第二个弹窗中确认（记录消失）
- **体验要点**：二次确认流程是否合理？撤销后反馈是否及时？

### 2.5 管理员总览 Tab（平台管理员可见）

- 查看全平台组织数 / 用户数 / 组织 usage 表格
- **体验要点**：数字是否和你预期相符？表格是否易读？

---

## Step 3 · 组织配额试用

1. 进入「我的组织」（或通过 Org 面板）
2. 查看配额使用情况（Members / Lakes / Nodes）
3. 尝试邀请一个成员（输入已注册邮箱），确认成员数 +1
4. **体验要点**：配额展示是否直观？是否知道自己还剩多少配额？

---

## Step 4 · 节点创建与全文检索

1. 进入一个湖，创建 2-3 个节点，内容随意
2. 按 `Cmd/Ctrl + K` 打开搜索，搜索刚创建节点的关键词
3. **体验要点**：搜索响应速度？结果是否准确？
