# Phase 13 全文检索 10k 基线（2026-04-27）

## 环境

- 目标环境：`http://fn.cky:18000`
- 数据范围：专用测试 lake，累计导入 `10,000` 个 TEXT 节点
- 导入方式：`10` 批次 `x 1000`，调用 `POST /api/v1/lakes/{id}/nodes/batch`

## 验收步骤

1. 注册临时用户并登录，创建专用 lake；
2. 分 10 批导入 10,000 个节点，每个节点内容包含公共检索词 `searchbaseline20260427` 与唯一 marker；
3. 轮询 `GET /api/v1/search?q=marker10000&lake_id=<lakeId>&limit=5`，直到最后一批 marker 可被检索；
4. 对 `GET /api/v1/search?q=searchbaseline20260427&lake_id=<lakeId>&limit=20` 连续采样 `50` 次，计算延迟分位。

## 结果

- 批次导入：`10 / 10` 成功；
- readiness gate：成功，最后一批 marker 可检索；
- 样本数：`50`；
- 每次请求均返回至少 `1` 条结果；
- `p50 = 41.2039ms`；
- `p95 = 66.4085ms`；
- `p99 = 117.9482ms`。

## 结论

在 `10k` 节点规模下，全文检索接口满足 Phase 13 准入门槛 `p95 < 300ms`。该项现可从“待造数与采样”更新为“已验收通过”。
