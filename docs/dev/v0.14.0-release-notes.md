## 亮点总览

Phase 14 将 Ripple 从「功能完整」推进到「**可运营、可管理、可准入**」——组织配额计量全链路闭环、运营管理台上线、灰度名单 CRUD、平台管理员角色体系，以及去 CI 化的本地准入门替代方案。

## 新功能

### 组织配额全链路（14.1 + 14.2）

- **配额数据模型**：`org_quotas` / `org_usage_snapshots` 表，支持免费套餐默认限额（湖 50 / 成员 3 / 节点 10000 / 附件 100 / 存储 512MB / API Key 10）。
- **配额拦截**：以下操作超限均返回 `429`，客户端收到明确错误提示：
  - 建湖（`SetLakeOrg`）
  - 添加成员（`AddMember`）
  - 上传附件（`UploadAttachment`）
  - 创建/批量导入节点（`CreateNode` / `BatchImportNodes`）
  - 创建 API Key（绑定 Org 时）
- **Condense 防重计**：同湖/同组织内 Condense 生成的 PermaNode delta=0，不重复累计配额。
- **API**：`GET/PATCH /api/v1/organizations/{id}/quota`。

### 运营管理台（14.3）

- **Org 列表增强**：直接展示 Members / Lakes / Nodes 用量摘要与最近配额审计记录。
- **组织总览 API**：
  - `GET /api/v1/organizations/{id}/overview`（单 Org 详情）
  - `GET /api/v1/organizations/overview`（平台管理员全局视图）
- **Settings 页面**：平台管理员可在 Settings → 管理员总览查看全平台组织数 / 用户数 / 组织 usage 表格。

### 灰度名单管理（14.3 + 14.6）

- 后端 `graylist_entries` 表 + `GET/POST /api/v1/admin/graylist` + `DELETE /api/v1/admin/graylist/{id}`。
- 注册流程灰度校验：非灰度邮箱尝试注册返回 403。
- Settings → 灰度名单 Tab：平台管理员可新增/删除邮箱，并附备注。

### 平台管理员角色管理（14.6）

- `GET/POST /api/v1/admin/platform_admins` + `DELETE /api/v1/admin/platform_admins/{id}`。
- Settings → 平台管理员 Tab：展示当前管理员列表，支持新增与 OWNER 权限撤销（含二次确认弹窗）。
- E2E 覆盖：graylist 增删（含确认弹窗）+ OWNER 撤销（dismiss 保留 / accept 移除 两轮确认）。

## 工程改进

### 准入体系去 CI 化（Phase 14.6）

- 移除 `.github/workflows/ci.yml`（GitHub Actions 收费），全面切换为本地脚本门禁：
  - `go test -race -count=1 ./...`（10 包，最长 7.4s，0 race 0 fail）
  - `go vet ./...`
  - `npm.cmd run lint / test / build`
  - Playwright E2E：`settings-tabs.spec.ts` 3/3 PASS
- 新增一键准入脚本 `scripts/smoke/phase14-6-acceptance.ps1`：
  - 自动 transcript 时间戳归档 → `docs/launch/acceptance-logs/`
  - 可选 `-IncludeStagingCleanup` 串入 staging smoke 数据清理

### Staging Smoke 数据清理

- `scripts/smoke/staging-cleanup-smoke.ps1`：SSH + `docker exec` 方式清理 staging PG（users + 关联 RESTRICT FK 链路）与 Neo4j smoke 数据；默认 dry-run，`-Apply` 触发真实删除；事务 `BEGIN/COMMIT` 安全包裹。

